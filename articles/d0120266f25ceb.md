## はじめに

機械学習モデルは、学習時に見たデータと同じ分布から得られるデータに対しては高い性能を示すことがある。しかし、実際の応用では、モデルが学習時には見ていない環境に展開されることも多い。カメラトラップ画像、病理画像、鳥類音声などでは、撮影場所、病院、録音機器、地域といった違いによって、学習時とは異なるドメインシフトが自然に発生する。

このような未知ドメインへの汎化は、**out-of-domain**、あるいは **OOD 汎化** と呼ばれる。OOD 汎化を改善するための代表的な方法のひとつが、データ拡張である。画像を切り抜く、反転する、色を変える、サンプル同士を混合するなどの操作によって、モデルが表面的な変化に過度に依存しないようにする。

しかし、OOD 汎化においては、単にデータを多様化すればよいわけではない。問題は、どの特徴を変化させ、どの特徴を保持するかである。特に実世界のデータでは、ドメインごとに変化する特徴の中にも、予測に有用な情報が含まれている場合がある。

たとえば、カメラトラップ画像における背景は、カメラごとに異なるためドメイン依存的である。一方で、その背景には、生息環境に関する情報も含まれている。サバンナなのか、森林なのか、水辺なのかといった環境の違いは、どの動物が現れやすいかを予測するうえで有用な手がかりになりうる。

したがって、OOD 汎化のためにドメイン依存特徴をすべて消す、という方針は必ずしも適切ではない。重要なのは、未知ドメインでも有用な特徴と、訓練ドメイン内でだけ偶然ラベルと相関している特徴を **区別する** ことである。

本記事で扱う *Out-of-Domain Robustness via Targeted Augmentations* は、この視点から OOD 汎化のためのデータ拡張を考える論文である。提案される Targeted Augmentations は、疑似相関的なドメイン依存特徴を選択的にランダム化しつつ、予測に有用なドメイン依存特徴は保持することを目指す。

この記事では、OOD 汎化におけるデータ拡張の問題設定、特徴分解の考え方、Targeted Augmentations の設計、理論解析の直感、そして実世界データセットでの実験結果を順に整理する。

## 1. OOD汎化における「データ拡張」の問題設定

### 1.1 未知ドメインで性能が落ちる理由

実世界の機械学習では、学習時に観測したデータと、実際にモデルを適用するデータが同じ分布に従うとは限らない。むしろ、モデルは学習時には見ていない環境に展開されることが多い。

たとえば、野生動物モニタリングのためのカメラトラップ画像を考える。この設定では、固定式カメラで撮影された画像から、写っている動物種を分類する。学習時にはいくつかのカメラから得られた画像を用いるが、実際の運用では、学習時には含まれていなかった新しいカメラの画像に対して予測する必要がある。

このとき、新しいカメラでは背景、撮影角度、照明条件、出現する動物の分布などが変化しうる。学習時のカメラでは高い性能を示していたモデルであっても、特定のカメラに固有の背景や撮影条件に依存していれば、未知のカメラでは性能が低下する。

同様の問題は、病理画像や鳥類音声でも生じる。病理画像では、病院や撮像条件の違いによって染色色調や患者集団が変化する。鳥類音声では、マイクの種類、録音環境、地域ごとの背景音が変化する。これらはいずれも、学習時のドメインと評価時のドメインが異なることによって生じる分布シフトである。

ここで重要なのは、この問題が単なるテストデータの違いではないという点である。モデルは、有限個の訓練ドメインから、まだ観測していない未知ドメインへ一般化しなければならない。したがって、訓練データ上で高い性能を得るだけでは不十分である。訓練ドメインに固有の手がかりではなく、未知ドメインでも通用する規則や特徴に基づいて予測する必要がある。

このような問題が、out-of-domain、すなわち OOD 汎化の問題である。

### 1.2 既存のデータ拡張では何が足りないのか

OOD汎化を改善するための代表的な方法のひとつが、データ拡張である。データ拡張では、学習データに人工的な変換を加えることで、モデルが表面的な変化に過度に依存しないようにする。

画像分類では、ランダムクロップ、水平反転、色変換、RandAugment、MixUp、CutMix などがよく使われる。これらの一般的なデータ拡張は、イン・ドメイン性能の改善に有効であることが多い。モデルが細かな見た目の違いに対して頑健になり、学習データへの過剰適合を抑えられるためである。

しかし、OOD汎化の文脈では、一般的なデータ拡張が常に十分であるとは限らない。なぜなら、**未知ドメインで問題となる分布シフトは、単なる画像の切り抜き方や反転、局所的な見た目の変化だけでは説明できない** からである。

たとえば、カメラトラップ画像にランダムクロップや水平反転を適用しても、特定のカメラに固有の背景は残り続ける可能性がある。病理画像でも、一般的な画像変換だけでは、病院ごとの染色色調に由来する偏りを十分に取り除けないかもしれない。鳥類音声でも、通常のスペクトログラム変換だけでは、マイクや録音地点に固有の手がかりを壊せない場合がある。

つまり、一般的なデータ拡張は、サンプル単位の見た目の多様性を増やすことには役立つ。しかし、ドメインごとに系統的に変化する特徴を適切に扱えるとは限らない。OOD汎化に必要なのは、単にデータを増やすことではなく、**未知ドメインで破綻しやすい手がかりへの依存を減らすこと** である。

一方で、別の方向性として、ドメインに依存する特徴をできるだけ消し、ドメイン不変な表現を学習しようとする考え方がある。これは直感的には自然である。未知ドメインへ汎化したいのであれば、ドメインごとに変わる特徴ではなく、ドメインをまたいで安定した特徴に依存すべきだと考えられるからである。

しかし、この方針にも限界がある。実世界のデータでは、ドメインごとに変化する特徴の中にも、予測に有用な情報が含まれている場合がある。したがって、ドメイン依存特徴をすべて消そうとすると、未知ドメインでも役立つ情報まで失われる可能性がある。

この点が、OOD汎化におけるデータ拡張を難しくしている。一般的なデータ拡張では、ドメインに固有の疑似相関を十分に壊せない。一方で、ドメイン依存特徴をすべて壊すと、有用な情報まで壊してしまう。したがって、必要なのはその中間にある設計である。

### 1.3 本論文の中心的な問い

本論文の中心的な問いは、OOD汎化のために、ドメイン依存特徴をどのように扱うべきかである。

ひとつの立場は、ドメイン依存特徴をできるだけ捨てることである。カメラごとの背景、病院ごとの染色色調、マイクごとの録音条件などは、訓練ドメインに固有の手がかりであり、未知ドメインでは通用しない可能性がある。そのため、これらを取り除き、ドメイン不変な特徴だけに依存するモデルを学習する、という考え方である。

しかし、本論文はこの見方をそのまま受け入れない。なぜなら、ドメイン依存特徴の中には、未知ドメインでも予測に有用な情報が含まれている場合があるからである。

カメラトラップ画像の背景は、たしかにカメラごとに変化する。しかし、その背景には生息環境の情報も含まれる。サバンナ、森林、水辺といった環境の違いは、どの動物が現れやすいかを予測するうえで有用である。つまり、背景はドメイン依存的でありながら、同時に予測に役立つ特徴でもある。

したがって、問題は「ドメイン依存特徴を使うか、使わないか」ではない。より重要なのは、**ドメイン依存特徴の中で、捨てるべきものと残すべきものを区別すること** である。

訓練ドメイン内でだけ偶然ラベルと相関している特徴には依存すべきではない。一方で、ドメインごとに変化していても、未知ドメインで予測に有用な特徴は保持すべきである。OOD汎化のためのデータ拡張は、この区別に基づいて設計される必要がある。

本論文で提案される Targeted Augmentations は、この立場に基づく。Targeted Augmentations は、ドメイン依存特徴を一括して壊すのではなく、疑似相関的な特徴を選択的にランダム化し、予測に有用な特徴はできるだけ保持することを目指す。

言い換えると、本論文はデータ拡張を、単なる汎用的な正則化手法としてではなく、「どの情報を壊し、どの情報を残すか」を設計する操作として捉え直している。

次章では、この考え方をより明確にするために、本論文が導入する特徴分解を整理する。特に、ドメイン依存特徴をひとまとめに扱うのではなく、予測に有用なものと疑似相関的なものに分けて考える視点が重要になる。

## 1. OOD汎化における「データ拡張」の問題設定

* 実世界の機械学習では、訓練時に見ていないドメインへモデルを展開することが多い。

### 1.2 既存のデータ拡張では何が足りないのか

* RandAugment、MixUp、CutMix などの generic augmentation は、ID性能には効くことがあるが、OOD性能への効果は不安定。
* domain-invariant augmentation は、ドメイン依存特徴を消す方向に働く。

* それとも、捨てるべきドメイン依存特徴と、残すべきドメイン依存特徴を区別すべきなのか。
* 本論文は後者の立場から、targeted augmentations を提案する。

## 2. 問題設定：特徴を4種類に分解する

### 2.1 ドメイン汎化の定式化

* 学習ドメイン集合と未知ドメイン集合を区別する。

  * $x_{\text{noise}}$：ドメイン非依存だが予測に不要な特徴
  * $x_{\text{d:robust}}$：ドメイン依存だが予測に有用な特徴
  * $x_{\text{d:spu}}$：ドメイン依存かつ疑似相関的な特徴

* サンプル数を増やしても、ドメイン数が少なければこの問題は残る。
* ここで「サンプル複雑性」ではなく「ドメイン複雑性」の問題であることを強調する。

## 3. Targeted Augmentations：何を壊し、何を残すのか

### 3.1 3種類のデータ拡張の違い

* generic augmentations：$x_{\text{noise}}$ を主に変える。
* domain-invariant augmentations：$x_{\text{d:robust}}$ と $x_{\text{d:spu}}$ の両方を壊す。
* targeted augmentations：$x_{\text{d:spu}}$ だけを壊し、$x_{\text{d:robust}}$ は残す。
* 式(5)〜(7)を使うならこの節。

### 3.2 実世界データセットでの具体例

* IWILDCAM2020-WILDS：Copy-Paste (Same Y)

  * 動物の前景を、同じラベルが観測されるカメラ由来の背景へ貼る。
  * 低レベル背景は壊すが、生息環境情報はある程度残す。
* CAMELYON17-WILDS：Stain Color Jitter

  * 染色色調という疑似相関的特徴を壊す。
  * 細胞形態や病期に関連する情報は壊しすぎない。
* BIRDCALLS：Copy-Paste + Jitter (Region)

  * 鳥の鳴き声を同一地域内の録音へ貼る。
  * マイク依存ノイズは壊し、生息環境音は残す。

## 4. 理論解析：なぜ Targeted Augmentation は効くのか

### 4.1 線形回帰設定の直感

* 複雑な画像・音声問題を直接解析する代わりに、線形回帰の toy setting を導入する。
* 各ドメインが潜在属性 $\mu^{(d)}$ を持ち、それがドメイン依存特徴を生む。
* ラベルは $x_{\text{obj}}$ と $\mu^{(d)}*{\text{robust}}$ に依存し、$\mu^{(d)}*{\text{spu}}$ には依存しない。

### 4.2 underspecification：ドメイン数が足りないと何が起きるか

* 学習ドメイン数 $D$ がドメイン依存特徴の次元 $p_{\text{dom}}$ より小さいと、問題が underspecified になる。
* その結果、モデルは訓練ドメイン内で偶然成立した $x_{\text{d:spu}}$ とラベルの相関を使ってしまう。
* generic augmentation はこの問題を解決しない。なぜなら、ドメイン依存特徴そのものを十分にランダム化しないから。

### 4.3 Theorem 1〜4 の読み方

* Theorem 1：データ拡張なしでは、$D < p_{\text{dom}}$ のとき OOD リスクが下がりにくい。
* Theorem 2, 3：targeted augmentation は有効次元を $p_{\text{dom}}$ から $p_{\text{robust}}$ に下げ、より少ないドメイン数で汎化しやすくする。
* Theorem 4：domain-invariant augmentation は、ロバストなドメイン依存特徴まで壊すため、高い OOD 誤差を生む。
* この節では証明の詳細よりも、境界が何を意味しているかを説明する。

## 5. 実験結果：実世界でも本当に効くのか

### 5.1 実験設定と比較手法

* 3つの実世界データセットを扱う。

  * IWILDCAM2020-WILDS
  * CAMELYON17-WILDS
  * BIRDCALLS
* 比較対象は以下。

  * データ拡張なし
  * generic augmentations
  * domain-invariance 系手法
  * targeted augmentations
* ここでは個々の実装詳細よりも、比較の軸を明確にする。

### 5.2 主結果：Targeted Augmentations は OOD 性能を改善する

* Figure 4 を中心に解説する。
* iWildCam、Camelyon17、BirdCalls のすべてで targeted augmentations が最良または大きく改善。
* 特に Camelyon17 では改善幅が大きく、論文の主張を強く支える結果になっている。
* ID性能の改善ではなく、同程度のID性能でもOOD性能が上がる点を強調する。

### 5.3 アブレーションと CLIP finetuning

* iWildCam と BirdCalls では、ロバストなドメイン依存特徴を保持しないアブレーションで性能が下がる。
* これは、$x_{\text{d:robust}}$ を残すことが本当に重要であることを示している。
* CLIP ViT-L/14 の finetuning でも targeted augmentations が改善をもたらす。
* ただし、大規模事前学習モデルでも改善幅は限定的なので、過度に一般化しない。

## 6. まとめと批判的考察

### 6.1 この論文の貢献

* OOD汎化におけるデータ拡張を、「何を壊し、何を残すか」という観点から整理した。
* ドメイン依存特徴をすべて悪者にせず、ロバストなドメイン依存特徴を明示的に扱った。
* 理論解析と実世界実験の両方で、targeted augmentations の有効性を示した。

### 6.2 この論文の強み

* 問題設定が実世界に近い。
* domain invariance の素朴な前提に対して、明確な反例的視点を与えている。
* 手法自体は単純だが、設計原理が明確で応用可能性がある。

### 6.3 注意点と限界

* targeted augmentation の設計には強いドメイン知識が必要。
* どの特徴が $x_{\text{d:robust}}$ で、どの特徴が $x_{\text{d:spu}}$ かを自動で判定しているわけではない。
* データセットごとに手作業で設計された augmentation なので、完全に汎用的な手法とは言いにくい。
* むしろ本論文の価値は、万能アルゴリズムの提案ではなく、OOD向けデータ拡張設計の考え方を与えた点にある。
