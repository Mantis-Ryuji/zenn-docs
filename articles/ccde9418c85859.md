## Just Train Twice 論文解説：失敗例を重く見るだけで group robustness を改善するシンプルな手法

機械学習モデルの性能を評価するとき、最もよく用いられる指標は平均精度である。訓練データ全体、あるいはテストデータ全体に対して、どれだけ正しく予測できたかを見る。この評価は単純で分かりやすく、多くの場面で有用である。

しかし、平均精度が高いことは、すべてのデータ群に対して安定して高い性能を示すことを意味しない。特に、入力とラベルの間に **疑似相関** が存在する場合、ERM によって学習されたモデルは、その疑似相関に依存することで平均的には高い精度を達成しつつ、特定のグループに対して大きく失敗することがある。

例えば、鳥の分類において「水鳥は水辺にいる」「陸鳥は陸地にいる」という相関が訓練データに強く存在している場合、モデルは鳥そのものではなく背景に依存して分類する可能性がある。このとき、陸地にいる水鳥や水辺にいる陸鳥のように、訓練データ中では少数派であるサンプルに対して性能が大きく低下しうる。平均精度だけを見ていると、このような失敗は見えにくい。

この問題に対して、**Group DRO** は各グループの損失を考慮し、最も性能が悪いグループの損失を直接小さくする手法である。これは worst-group performance を改善するうえで強力な方法である。一方で、Group DRO には大きな制約がある。各訓練サンプルがどのグループに属するかを示す **グループアノテーション** が必要になるのである。

**Just Train Twice (JTT)** は、この制約を緩和するために提案された非常に単純な手法である。JTT では、まず通常の ERM モデルを短く学習し、そのモデルが誤分類した訓練サンプルを特定する。次に、その誤分類されたサンプルの重みを大きくして、もう一度モデルを学習する。名前の通り、やることは **「二回学習する」だけ** である。

一見すると、これは単なる失敗例の再重み付けに見える。しかし重要なのは、ERM が失敗したサンプル集合が、単なるノイズの集まりではなく、疑似相関のもとで性能が崩れやすいグループを反映している可能性があるという点である。つまり、JTT は訓練グループアノテーションを直接使わずに、ERM の失敗を手がかりとして group robustness を改善しようとする手法である。

本記事では、JTT のアルゴリズムそのものに加えて、原論文で比較されている ERM、CVaR DRO、Learning from Failure（LfF）、Group DRO との関係を整理する。そのうえで、JTT がなぜ機能するのか、なぜ単に高損失サンプルを動的に重み付けする CVaR DRO とは異なるのか、そして JTT がどのような限界を持つのかを見ていく。

## 1. Setup：group robustness と疑似相関

### 1.1 ERM が見ているもの

まず、標準的な経験リスク最小化（empirical risk minimization; ERM）が何を最適化しているのかを確認する。

入力を $x \in \mathcal{X}$、ラベルを $y \in \mathcal{Y}$ とする。訓練データとして、$n$ 個のサンプル $\{(x_1, y_1), \ldots, (x_n, y_n)\}$ が与えられているとする。また、モデルを $f_\theta : \mathcal{X} \to \mathcal{Y}$ とし、$\theta \in \Theta$ をそのパラメータとする。損失関数を $\ell(x,y;\theta)$ と書けば、ERM は次の平均訓練損失を最小化する。

$$
J_{\mathrm{ERM}}(\theta)=\frac{1}{n}\sum_{i=1}^{n}\ell(x_i,y_i;\theta)
$$

この目的関数は、訓練データ全体に対する平均損失を小さくすることを目指している。したがって、データセット中で多数を占めるサンプルに対して性能が高ければ、全体の平均精度も高くなりやすい。

しかし、平均損失を最小化することは、すべての部分集団に対して均等に良い性能を出すことを意味しない。あるグループのサンプル数が少ない場合、そのグループで大きな誤差が生じていても、全体平均に対する寄与は小さい。そのため、ERM は平均的には良いモデルを返す一方で、特定のグループでは大きく失敗することがある。

この問題は、入力とラベルの間に **疑似相関** が存在する場合に特に顕著になる。モデルは、タスクに本質的な特徴ではなく、訓練データ中でたまたまラベルと相関している特徴を使って分類できてしまうからである。

例えば、鳥の分類タスクで、水鳥の画像には水辺の背景が多く、陸鳥の画像には陸地の背景が多いとする。このとき、モデルは鳥の形状や種類そのものではなく、背景を使って分類しても、訓練データ上では高い精度を達成できる。しかし、そのように学習されたモデルは、陸地にいる水鳥や水辺にいる陸鳥に対して失敗しやすい。

つまり、ERM の問題は単に「性能が低い」ことではない。 **平均精度だけを見ると高性能に見えるにもかかわらず、特定のグループで構造的に失敗することが問題なのである。**

### 1.2 worst-group error

このような失敗を捉えるために、本論文では平均精度ではなく、worst-group performance に注目する。

あらかじめ定義されたグループの集合を $\mathcal{G}$ とする。各サンプル $(x,y)$ は、あるグループ $g \in \mathcal{G}$ に属しているとする。このとき、分類器の性能を次の worst-group error で評価する。

$$
\max_{g \in \mathcal{G}}\mathbb{E}[\ell_{0\text{-}1}(x,y;\theta)\mid g]
$$

ここで、$\ell_{0\text{-}1}(x,y;\theta)$ は $0$-$1$ loss であり、

$$
\ell_{0\text{-}1}(x,y;\theta)=\mathbf{1}[f_\theta(x) \neq y]
$$

で定義される。すなわち、モデル $f_\theta$ が入力 $x$ に対して正しいラベル $y$ を予測できなかった場合に $1$、正しく予測できた場合に $0$ を返す損失である。

この指標は、各グループにおける分類誤差を計算し、その中で最も大きいものを見る。したがって、全体の平均精度が高くても、どこか一つのグループで性能が大きく崩れていれば、その失敗が直接評価に反映される。

ここで重要なのは、worst-group error が平均性能とは異なる評価軸を持っていることである。ERM が最小化するのは、全サンプルにわたる平均損失である。一方、worst-group error が見るのは、最も性能が悪いグループである。

この違いは、モデル選択においても重要になる。平均 validation accuracy に基づいてモデルを選ぶことと、worst-group validation accuracy に基づいてモデルを選ぶことは、一般には同じではない。平均精度を最大化するモデルが、必ずしも worst-group performance に優れるとは限らない。

JTT が扱うのは、この worst-group performance を改善する問題である。ただし、ここで重要な制約がある。訓練データ全体に対して、各サンプルがどのグループに属するかを示す **グループアノテーション** は与えられていない、という設定である。

つまり、本論文の目標は次のように整理できる。

$$
\begin{aligned}
&\textbf{訓練グループアノテーションなしで、}\\
&\textbf{テスト時の worst-group error を小さくする。}
\end{aligned}
$$

ただし、ハイパーパラメータ調整のために、小規模な **検証セットにはグループアノテーションが付与されている** と仮定する。この検証セットを用いることで、worst-group validation error を計算し、モデル選択やハイパーパラメータ調整を行う。

この点は、JTT を理解するうえで非常に重要である。JTT は訓練グループアノテーションを **使わない** 手法である。しかし、完全にグループ情報を不要にする手法ではない。少なくとも原論文の設定では、小規模な検証セット上のグループアノテーションを用いて、worst-group performance に基づくモデル選択を行う。

:::message alert
なお、ここでの「小規模」は、worst-group performance の推定が必ず安定することを意味しない。検証セット中の各 group のサンプル数が少なければ、worst-group performance に基づくモデル選択は当然不安定になり得る。JTT の利点は、group annotation を不要にすることではなく、訓練データ **全体** への annotation を必要とせず、検証セット上の annotation に **限定できる** 点にある。
:::

### 1.3 疑似相関によるグループ

本論文が主に扱うのは、ラベル $y$ と疑似属性 $a$ によってグループが定義される設定である。

疑似属性を $a \in \mathcal{A}$ とする。ここで、$\mathcal{A}$ は疑似属性の取りうる値の集合である。例えば Waterbirds であれば、疑似属性は背景であり、水背景か陸背景かを表す。

このとき、各グループ $g$ は、疑似属性 $a$ とラベル $y$ の組として定義される。

$$
g = (a, y)
$$

したがって、グループ集合は次のように表される。

$$
\mathcal{G}=\mathcal{A}\times\mathcal{Y}
$$

この定義により、単に「水背景の画像」や「waterbird の画像」を見るのではなく、「水背景の waterbird」「陸背景の waterbird」「水背景の landbird」「陸背景の landbird」のように、疑似属性とラベルの組み合わせごとに性能を評価できる。

Waterbirds の例で考えると、ラベル $y$ は waterbird または landbird であり、疑似属性 $a$ は背景が water か land かである。このとき、グループは次の4つになる。

* water background 上の waterbird
* land background 上の waterbird
* water background 上の landbird
* land background 上の landbird

訓練データでは、通常、waterbird は water background と、landbird は land background と強く相関している。つまり、多くの訓練サンプルでは、鳥の種類と背景が一致している。

![](https://static.zenn.studio/user-upload/0a8b102cb02c-20260603.png)
*Figure 1. Waterbirds データセットにおける4つのグループ*

このような状況では、背景はラベル予測に使える強い手がかりになる。しかし、それはタスクに本質的な特徴とは限らない。背景と鳥の種類の相関は、訓練データ中では成り立っていても、テスト時に常に成り立つとは限らないからである。

このように、訓練データ中ではラベルと相関しているが、予測対象そのものを本質的に決めるとは限らない属性を、ここでは疑似属性と呼ぶ。そして、その疑似属性とラベルの相関を疑似相関と呼ぶ。

疑似相関が存在すると、ERM は多数派グループに適合しやすくなる。Waterbirds でいえば、水背景の waterbird や陸背景の landbird は分類しやすい。一方で、陸背景の waterbird や水背景の landbird は、訓練データ中で少数派になりやすく、モデルが背景に依存している場合には誤分類されやすい。

したがって、この設定における group robustness の問題は、次のように言える。

$$
\begin{aligned}
&\textbf{疑似相関に依存して平均精度を上げるのではなく、}\\
&\textbf{疑似相関が成り立たないグループでも性能を保てるか。}
\end{aligned}
$$

JTT は、この問題に対して、訓練グループアノテーションを使わずに取り組む。次章では、その位置づけを明確にするために、原論文で比較されている ERM、CVaR DRO、LfF、Group DRO を整理する。

## 2. 比較手法：ERM / CVaR DRO / LfF / Group DRO

JTT を理解するには、単にアルゴリズムの手順を見るだけでは不十分である。重要なのは、JTT がどのような既存手法の間に位置しているのかである。

原論文では、JTT の比較対象として主に4つの手法が整理されている。ERM、CVaR DRO[^1]、Learning from Failure (LfF)[^2] 、Group DRO[^3] である。これらはそれぞれ、JTT の異なる側面を理解するための基準になっている。

ERM は、平均損失を最小化する標準的なベースラインである。CVaR DRO は、訓練グループアノテーションなしで高損失サンプルを重く見る手法であり、JTT と表面的に似ている。LfF は、一つ目のモデルの失敗を利用して二つ目のモデルを学習するという点で、JTT に最も近い比較対象である。Group DRO は、訓練グループアノテーションを用いて worst-group loss を直接最小化する手法であり、JTT がどこまで近づけるかを見る oracle 的な基準になる。

この章では、これら4つの手法を順に整理し、JTT の位置づけを明確にする。

[^1]: Daniel Levy, Yair Carmon, John C. Duchi, and Aaron Sidford, “Large-Scale Methods for Distributionally Robust Optimization,” NeurIPS 2020. [https://arxiv.org/abs/2010.05893](https://arxiv.org/abs/2010.05893)
[^2]: Junhyun Nam, Hyuntak Cha, Sungsoo Ahn, Jaeho Lee, and Jinwoo Shin, “Learning from Failure: Training Debiased Classifier from Biased Classifier,” NeurIPS 2020. [https://arxiv.org/abs/2007.02561](https://arxiv.org/abs/2007.02561)
[^3]: Shiori Sagawa, Pang Wei Koh, Tatsunori B. Hashimoto, and Percy Liang, “Distributionally Robust Neural Networks for Group Shifts: On the Importance of Regularization for Worst-Case Generalization,” ICLR 2020. [https://arxiv.org/abs/1911.08731](https://arxiv.org/abs/1911.08731)

### 2.1 ERM：平均損失を最小化する標準手法

ERM は、訓練データ全体に対する平均損失を最小化する標準的な学習方法である。損失関数を $\ell(x,y;\theta)$ とすると、ERM の目的関数は次のように書ける。

$$
J_{\mathrm{ERM}}(\theta)=\frac{1}{n}\sum_{i=1}^{n}\ell(x_i,y_i;\theta)
$$

ERM の利点は、その単純さにある。グループ情報も、疑似属性の情報も、特別な再重み付けも必要としない。通常の教師あり学習として、平均訓練損失を下げればよい。

しかし、第1章で述べたように、ERM は平均損失を見ている。そのため、データセット中で多数を占めるグループに対して性能が高ければ、少数派グループで大きな誤差が生じていても、全体の平均損失は十分に小さくなりうる。疑似相関がある場合、この問題はより顕著になる。モデルがラベルそのものに対応する本質的特徴ではなく、ラベルと相関した疑似属性を使って予測しても、多数派グループでは高い精度を達成できるからである。

したがって、ERM は JTT にとって二重の意味を持つ。一つは、改善すべき標準ベースラインである。もう一つは、JTT の内部で実際に使われる学習手続きでもある。JTT は ERM を捨てる手法ではない。むしろ、ERM を一度学習し、その失敗を利用して、もう一度 ERM を行う手法である。

この意味で、JTT は ERM の対立物ではなく、ERM の失敗を情報として再利用する方法である。

### 2.2 CVaR DRO：高損失サンプルを動的に重く見る手法

CVaR DRO は、訓練グループアノテーションを用いずに worst-group performance を改善しようとする自然なアプローチである。

DRO、すなわち distributionally robust optimization では、経験分布そのものに対する平均損失ではなく、経験分布の周辺にある不確実性集合内の最悪ケース分布に対する損失を小さくしようとする。CVaR は、その代表的な目的関数の一つである。

レベル $\alpha \in (0,1]$ の CVaR 目的関数は、次のように書ける。

$$
J_{\text{CVaR}}(\theta,\alpha)=\sup_{q\in\Delta^n}\Biggl\lbrace \sum_{i=1}^n q_i \ell(x_i,y_i;\theta) \; \text{s.t.} \; \|q\|_{\infty} \leq \frac{1}{\alpha n} \Biggr\rbrace
$$

ここで、$\Delta^n$ は $n$ 個の訓練サンプル上の確率単体である。$q_i$ は各訓練サンプルに割り当てる重みであり、制約 $\|q\|_{\infty} \leq 1/(\alpha n)$ によって、一部のサンプルに過度に重みが集中しすぎないようにしている。

この目的関数は、直感的には、損失が最も大きい $\alpha$-fraction の訓練サンプルに対する平均損失を見るものとして理解できる。つまり、全体平均ではなく、高損失なサンプル群に注目する。

:::details CVaR DRO における q の意味
補足すると、ここでの $q_i$ は手で決めるハイパーパラメータではない。$\theta$ を固定したとき、損失が大きくなるように **adversarial** に選ばれるサンプル重みである。したがって CVaR DRO は、現在のモデルが苦手としている高損失サンプルに重みを寄せ、その部分の損失を下げるように学習する目的関数だと見なせる。
:::

この発想は、JTT とよく似ている。JTT もまた、通常の ERM で失敗したサンプルを重く見るからである。どちらも訓練グループアノテーションを必要とせず、高損失サンプルを通じて poor group performance に対処しようとする。

しかし、両者には重要な違いがある。CVaR DRO は、現在のモデルにとって損失が大きいサンプル集合を、学習中に **動的に更新する。** 一方で、JTT は、最初の識別モデルが誤分類したサンプル集合を一度だけ作り、その集合を **固定する。**

この違いは、原論文では非常に重要な差分として扱われている。CVaR DRO は「今のモデルが苦手な点」を追い続ける。一方、JTT は「初期の ERM が失敗した点」を固定して重く見る。

後の実験で見るように、この固定された error set が JTT の性能にとって重要である。単に高損失サンプルを重く見ればよいわけではない。どの時点の、どのモデルにとっての失敗を使うかが重要なのである。

### 2.3 LfF：失敗から学ぶが、JTT より複雑な手法

Learning from Failure（LfF）は、JTT と概念的に近い手法である。どちらも、一つ目のモデルの失敗を利用して、二つ目のモデルの学習を補助するという構造を持つ。

LfF の基本的な考え方は、意図的にバイアスしたモデルを学習し、そのモデルが苦手とするサンプルを難しいサンプルとして扱うことである。疑似相関が存在する場合、バイアスしたモデルは学習しやすい疑似属性に依存しやすくなる。その結果、疑似相関が成り立たない少数派グループのサンプルに対して失敗しやすくなる。この失敗を利用して、二つ目のモデルを再重み付けする。

LfF では、二つのモデルを考える。一つ目は biased model、二つ目は debiased model である。biased model を $f_B$、debiased model を $f_D$ と書く。

biased model は、generalized cross-entropy（GCE）loss を用いて学習される。

$$
\ell_{\text{GCE}}(x_i,y_i;\theta_B,q)=\frac{1-f_B(y_i\mid x_i;\theta_B)^q}{q}
$$

ここで、$q \in [0,1)$ はハイパーパラメータである。GCE loss は、標準的な cross-entropy loss とは異なる勾配の性質を持ち、学習しやすいサンプルを相対的に重視する。そのため、biased model は、簡単な疑似相関に依存する方向へ誘導されやすい。

:::details GCE loss は何をしているのか
ここで

$$
p_i = f_B(y_i \mid x_i;\theta_B)
$$

とおくと、$p_i$ は biased model が入力 $x_i$ に対して正解クラス $y_i$ に割り当てる確率である。したがって、$p_i$ が大きいほどモデルは正解クラスを強く予測しており、$p_i$ が小さいほど正解クラスをうまく予測できていない。

GCE loss は

$$
\ell_{\text{GCE}}(x_i,y_i;\theta_B,q)=\frac{1-p_i^q}{q}
$$

で定義される。ここで $1-p_i^q$ は、正解クラスへの予測確率を「外している度合い」に変換している。実際、$p_i=1$ なら損失は $0$ になり、$p_i$ が小さくなるほど損失は大きくなる。

通常の cross entropy は $-\log p_i$ であり、$p_i \to 0$ のとき損失が無限大に発散する。一方、GCE loss は

$$
0 \leq \ell_{\text{GCE}} \leq \frac{1}{q}
$$

となり、損失が上に有界である。したがって、正解確率が非常に低い難しい例に対して、cross entropy ほど極端に大きな損失を与えない。

この性質により、GCE loss で学習された biased model は、難しい例を強く追いかけるよりも、早い段階で正解しやすい簡単な例に寄りやすくなる。疑似相関が本質的特徴よりも学びやすい場合、そのような簡単な例は bias-aligned sample に対応しやすい。その結果、biased model は疑似相関を強く利用するモデルとして学習される。
:::

次に、debiased model は、biased model と自身の予測を用いて各サンプルを再重み付けしながら学習される。原論文で示されている重みは、次の形である。

$$
\mathcal{W}(x_i,y_i)=\frac{\log f_B(y_i\mid x_i)}{\log f_B(y_i\mid x_i)+\log f_D(y_i\mid x_i)}
$$

この重み付けにより、biased model がうまく扱えないサンプル、すなわち疑似相関が成り立たないサンプルや難しいグループに属するサンプルが相対的に重く扱われることが期待される。

:::details LfF における重みの直感
LfF では、debiased model を学習するときに次の重みを用いる。

$$
\mathcal{W}(x_i,y_i)=\frac{\log f_B(y_i\mid x_i)}{\log f_B(y_i\mid x_i)+\log f_D(y_i\mid x_i)}
$$

ここで $f_B(y_i\mid x_i)$ は biased model が正解クラスに割り当てる確率、$f_D(y_i\mid x_i)$ は debiased model が正解クラスに割り当てる確率である。

一見すると分かりにくいが、確率の対数は $0$ 以下なので、cross entropy loss

$$
\ell_B=-\log f_B(y_i\mid x_i),\qquad \ell_D=-\log f_D(y_i\mid x_i)
$$

を使って書き直すと、

$$
\mathcal{W}(x_i,y_i)=\frac{\ell_B}{\ell_B+\ell_D}
$$

となる。つまり、この重みは「biased model の損失が、biased model と debiased model の損失の合計に占める割合」と見なせる。

biased model がそのサンプルを簡単に分類できる場合、$\ell_B$ は小さくなるため、$\mathcal{W}(x_i,y_i)$ も小さくなる。逆に、biased model がそのサンプルをうまく分類できない場合、$\ell_B$ は大きくなり、$\mathcal{W}(x_i,y_i)$ も大きくなる。

LfF では、biased model を意図的に疑似相関へ寄せて学習させる。そのため、biased model が失敗しやすいサンプルは、疑似相関だけでは分類しにくい bias-conflicting sample である可能性が高い。上の重みは、このようなサンプルを debiased model の学習で **相対的に重く見る** ための仕組みである。
:::

LfF と JTT は、どちらも **「一つ目のモデルの失敗を使う」** という点では似ている。しかし、JTT は LfF よりもかなり単純である。

LfF では、biased model を意図的に作るために GCE loss を導入し、biased model と debiased model を組み合わせて学習する必要がある。また、二つのモデルを同時に扱うため、学習手続きも複雑になる。

一方、JTT では、特別な loss を導入しない。意図的にバイアスしたモデルを設計することもしない。標準的な ERM モデルを短く学習し、そのモデルが誤分類したサンプルを固定して重く見るだけである。

つまり、LfF は **「失敗しやすいモデルを意図的に作る」** 手法であり、JTT は **「通常の ERM の初期の失敗をそのまま使う」** 手法である。

この差は、JTT の重要な特徴である。JTT は、failure-based な再重み付け手法でありながら、学習手続きは非常に単純である。

### 2.4 Group DRO：訓練グループ情報を用いる oracle

Group DRO は、ここまでの手法とは異なり、訓練グループアノテーションを用いる。すなわち、各訓練サンプル $(x_i,y_i)$ について、そのサンプルがどのグループ $g_i$ に属するかが分かっていると仮定する。

Group DRO の目的は、訓練データ上の worst-group loss を直接最小化することである。グループ $g$ に属する訓練サンプル数を $n_g$ とすると、Group DRO の目的関数は次のように書ける。

$$
J_{\text{Group-DRO}}(\theta)=\max_{g\in\mathcal{G}}\frac{1}{n_g}\sum_{i\mid g_1=g}\ell(x_i,y_i;\theta)
$$

ERM が全サンプルの平均損失を見るのに対して、Group DRO はグループごとの平均損失を計算し、その最大値を小さくしようとする。したがって、Group DRO は worst-group performance を改善するための直接的な目的関数である。

この点で、Group DRO は非常に強力である。平均損失ではなく、最も性能が悪いグループの損失を明示的に最適化するため、疑似相関が成り立たない少数派グループにも学習信号を与えやすい。

しかし、その代償として、訓練グループアノテーションが必要になる。これは実応用では大きな制約である。例えば、すべての訓練サンプルに対して、背景属性、人口統計属性、測定条件、ドメイン、サブグループなどを付与する必要がある場合、そのコストは非常に大きい。

そのため、JTT の原論文では、Group DRO は訓練グループアノテーションなしの手法に期待できる性能の上界を与える oracle method として扱われる。つまり、JTT は Group DRO と同じ情報条件で競争しているわけではない。Group DRO は訓練グループ情報を使える手法であり、JTT はそれを使えない手法である。

この違いを踏まえると、JTT の評価で重要なのは、JTT が Group DRO を完全に上回るかどうかではない。むしろ、訓練グループアノテーションを使わないにもかかわらず、ERM と Group DRO の間にある worst-group accuracy の差をどこまで埋められるかである。

### 2.5 JTT の位置づけ

以上の4つの比較手法を整理すると、JTT の位置づけが見えてくる。

ERM は、最も標準的で単純な平均損失最小化である。しかし、疑似相関のもとでは worst-group performance が崩れうる。

CVaR DRO は、訓練グループアノテーションなしで高損失サンプルを重く見る手法である。JTT と同じく、グループ情報なしに poor group performance へ対処しようとするが、重み付け対象を動的に更新する点が異なる。

LfF は、一つ目のモデルの失敗を使って二つ目のモデルを学習する点で JTT に近い。しかし、LfF は意図的にバイアスしたモデルや交互更新を必要とするため、JTT よりも複雑である。

Group DRO は、訓練グループアノテーションを使って worst-group loss を直接最小化する手法である。JTT にとっては、同じ条件で比較するベースラインというより、訓練グループ情報があればどこまでできるかを示す oracle 的な基準である。

以上を踏まえると、JTT は **ERM の単純さを保ちながら、失敗例に基づく重み付けを導入する手法** として位置づけられる。

JTT は、Group DRO のようにグループごとの損失を直接最小化するわけではない。また、CVaR DRO のように、現在のモデルが大きな損失を出しているサンプルを動的に追いかけるわけでもない。さらに、LfF のように、意図的に疑似相関へ寄せた biased model を設計するわけでもない。

JTT が行うのは、初期の ERM が誤分類した訓練サンプル集合を固定し、その集合を重くしてもう一度 ERM を行うことだけである。この単純さが、JTT の大きな特徴である。

次章では、この JTT の具体的なアルゴリズムを見ていく。

## 3. JTT：Just Train Twice

前章では、JTT の比較対象として ERM、CVaR DRO、LfF、Group DRO を整理した。ここからは、JTT そのものを見ていく。

JTT は、訓練時にグループアノテーションを使わずに worst-group performance を改善するための二段階手法である。名前の通り、基本的にはモデルを二回学習するだけである。

ただし、JTT の本質は単に「二回学習する」ことではない。重要なのは、一回目の ERM モデルが誤分類した訓練サンプルを error set として固定し、そのサンプル集合を重くして二回目の ERM を行う点である。

### 3.1 基本アイデア

JTT の基本的な流れは次の通りである。

1. 通常の ERM によって、識別モデルを短く学習する。
2. その識別モデルが誤分類した訓練サンプルを集め、error set を作る。
3. error set に含まれるサンプルを重くする。
4. 重み付けされた訓練データ上で、最終モデルをもう一度 ERM により学習する。

この手順は非常に単純である。特殊な損失関数を導入する必要はない。訓練データ全体に対するグループアノテーションも必要ない。最初のモデルを学習し、そのモデルが間違えたサンプルを重くして、もう一度学習するだけである。

JTT の直感は、ERM が失敗するサンプルには、疑似相関のもとで性能が崩れやすいグループの情報が含まれている、というものである。

疑似相関が存在するデータでは、ERM は学習しやすい相関に依存しやすい。例えば Waterbirds では、鳥そのものではなく背景を手がかりに分類できてしまう。その場合、背景とラベルの相関が成り立つ多数派グループでは高い精度を出せるが、相関が成り立たない少数派グループでは誤分類しやすくなる。

したがって、最初の ERM モデルが誤分類したサンプル集合には、worst-group examples が通常の訓練データよりも高い割合で含まれている可能性がある。JTT はこの性質を利用する。

### 3.2 Stage 1：識別モデルで error set を作る

JTT の第一段階では、訓練データ上で通常の ERM を行い、識別モデルを学習する。この識別モデルを $\hat{f}_{\mathrm{id}}$ と書く。

ただし、このモデルは最終的な分類器として使うためのものではない。目的は、高性能なモデルを得ることではなく、ERM がどの訓練サンプルで失敗するかを特定することである。

識別モデル $\hat{f}_{\mathrm{id}}$ を用いて、誤分類された訓練サンプルからなる集合 $E$ を定義する。

$$
E=\{(x_i,y_i)\;\text{s.t.}\;\hat{f}_{\text{id}}(x_i)\neq y_i\}
$$

この集合 $E$ が error set である。

ここで重要なのは、識別モデルを訓練データに完全に適合させないことである。もし識別モデルを長く学習しすぎると、訓練データ上の誤分類がほとんどなくなり、error set が空に近づいてしまう。その場合、どのサンプルを重くすべきかという情報が失われる。

そのため、JTT では識別モデルをあらかじめ決めた $T$ ステップだけ学習する。ここでの $T$ はハイパーパラメータであり、識別モデルをどの時点で打ち切るかを決める。

この打ち切りは、単なる計算コスト削減ではない。JTT が見たいのは、十分に学習した ERM が最終的にどこで失敗するかではなく、学習初期の ERM がどのサンプルをまだ分類できていないかである。言い換えると、JTT は「収束後の ERM の失敗」ではなく、**「短く学習した ERM の失敗」** を利用して error set を作る。

この点は、JTT の直感を理解するうえで重要である。学習初期の ERM は、まず学習しやすい相関に依存しやすい。そのため、疑似相関が成り立つ多数派グループには早く適合する一方で、疑似相関が成り立たないグループや少数派グループには適合しにくい。JTT は、その時点での誤分類集合を error set として取り出す。

### 3.3 Stage 2：error set を重くして再学習する

第二段階では、第一段階で得られた error set $E$ に含まれるサンプルの重みを大きくし、最終モデルを学習する。

重み付き ERM の目的関数は、次のように書ける。

$$
J_{\text{up-ERM}}(\theta,E)=\Biggl(\lambda_{\text{up}}\sum_{(x,y)\in E}\ell(x,y;\theta)+\sum_{(x,y)\notin E}\ell(x,y;\theta)\Biggr)
$$

ここで、$\lambda_{\mathrm{up}} \in \mathbb{R}_+$ は error set に含まれるサンプルをどれだけ重くするかを決めるハイパーパラメータである。

$\lambda_{\mathrm{up}}$ が大きいほど、第一段階で誤分類されたサンプルが二回目の学習で強く重視される。一方、$\lambda_{\mathrm{up}} = 1$ とすれば、error set のサンプルもそれ以外のサンプルも同じ重みで扱われるため、通常の ERM に戻る。

実装上は、この目的関数を明示的な重み付き損失として実装する必要はない。原論文では、error set に含まれる各サンプルを $\lambda_{\mathrm{up}}$ 回アップサンプリングし、それ以外のサンプルを一度だけ含むデータセットを作ることで実装している。

つまり、JTT の第二段階は、次のようなデータセット上で通常の ERM を行うことに対応する。

* error set に含まれるサンプル：$\lambda_{\mathrm{up}}$ 回出現させる
* error set に含まれないサンプル：1回だけ出現させる

このように見ると、JTT は実装としてもかなり単純である。特別な optimizer も、特殊なロバスト最適化手続きも必要ない。データのサンプリング比を変えたうえで、通常の ERM を行えばよい。

### 3.4 JTT のアルゴリズム

JTT の手順をまとめると、次のようになる。

:::message
**Input:** 訓練データ $\mathcal{D}$、識別モデルの学習ステップ数 $T$、重み付け係数 $\lambda_{\mathrm{up}}$

**Stage 1：identification**

1. 訓練データ $\mathcal{D}$ 上で、ERM により識別モデル $\hat{f}_{\mathrm{id}}$ を $T$ ステップ学習する。
2. 識別モデル $\hat{f}_{\mathrm{id}}$ が誤分類した訓練サンプルを集め、error set $E$ を構成する。

**Stage 2：upweighting**

3. error set に含まれるサンプルを $\lambda_{\mathrm{up}}$ 回、それ以外のサンプルを1回ずつ含むアップサンプリング済みデータセット $\mathcal{D}_{\mathrm{up}}$ を作る。
4. $\mathcal{D}_{\mathrm{up}}$ 上で、ERM により最終モデル $\hat{f}_{\mathrm{final}}$ を学習する。
:::

このアルゴリズムを見ると、JTT は非常に単純である。一回目のモデルで失敗例を見つけ、二回目のモデルでその失敗例を重く見るだけである。

ただし、この単純さは、JTT が何も仮定していないことを意味しない。**JTT は、初期の ERM が誤分類するサンプル集合に、worst-group performance 改善のために有用な情報が含まれている、という経験的性質に依存している。**

### 3.5 何が JTT らしさなのか

JTT の特徴を整理すると、重要な点は三つある。

1. **JTT は訓練グループアノテーションを使わない。**
   Group DRO のように、各訓練サンプルがどのグループに属するかを知っている必要はない。使うのは、通常の入力 $x_i$ とラベル $y_i$、そして識別モデルの誤分類情報だけである。

2. **JTT は特殊な失敗モデルを設計しない。**
   LfF のように、意図的にバイアスしたモデルを作るための特殊な loss を導入するわけではない。標準的な ERM モデルを短く学習し、その誤分類を使うだけである。

3. **JTT は error set を固定する。**
   これは CVaR DRO との重要な違いである。CVaR DRO は、現在のモデルにとって損失が大きいサンプル集合を学習中に動的に更新する。一方で JTT は、第一段階で得た error set を固定し、第二段階ではその集合を変えない。

このように、JTT はグループを直接推定するのではなく、短く学習した ERM の誤分類集合を手がかりとして使う。疑似相関が存在する状況では、この error set に、ERM が苦手とするグループのサンプルが **含まれやすい。** JTT はその集合を固定して重く見ることで、worst-group performance の改善を狙う。

次章では、この JTT が実験的にどの程度有効なのかを、ERM、CVaR DRO、LfF、Group DRO との比較を通じて見ていく。
