## Masked Autoencoders (MAE) 論文解説 ― なぜ高マスク率と非対称設計が効くのか

Masked Autoencoders（MAE）は、Vision Transformer 時代の自己教師あり学習を語るうえで外せない論文である。実際、この論文はしばしば「画像版 BERT」として紹介される。もちろん、その説明が完全に誤っているわけではない。入力の一部を隠し、隠した部分を予測するという枠組みだけを見れば、たしかに発想の表面はよく似ている。  
しかし、この理解だけでは MAE の本質をかなり取りこぼす。というのも、MAE の重要性は「画像でも masked modeling ができた」という一点にあるのではなく、 **画像という信号に合わせて masked modeling をどう設計し直したか** にあるからである。

自然言語と画像では、信号の性質が大きく異なる。言語の token は意味密度が高く、少数の欠損語を予測するだけでも高次の意味理解を要求しやすい。一方で画像は空間的冗長性が大きく、単純な欠損補完タスクをそのまま与えると、周囲の局所情報からそれらしく埋めるだけで解けてしまう場合がある。論文がまず問題にしているのは、まさにこの点である。すなわち、vision においては masked prediction をそのまま持ち込むだけでは、表現学習として十分に強い課題にならないのである。

そこで MAE は、非常に思い切った二つの設計を採用する。第一に、**75% 前後という高いマスク率** を用いて、再構成課題を意図的に難しくしたこと。第二に、 **エンコーダは可視パッチのみに作用し、再構成は軽量デコーダに任せる非対称設計** を採ったことである。この二つは独立な工夫ではない。高マスク率によって課題を難しくしつつ、同時に visible token しかエンコーダに入れないことで計算量を大きく削減する。MAE の美しさは、この「表現学習としての難しさ」と「計算効率としての軽さ」を同時に成立させた点にある。

本記事では、MAE を単なるアーキテクチャ紹介としては扱わない。見るべきなのは、各モジュールの名前や処理順そのものではなく、 **なぜその設計が必要だったのか** 、そして **どの設計選択がどの性能差として現れたのか** である。したがって、この記事の焦点は次の三点にある。  

- 第一に、なぜ vision では NLP と同じ感覚で masked modeling を設計できないのか。  
- 第二に、MAE の高マスク率と非対称 encoder-decoder 設計は、それぞれどの問題を解決しているのか。  
- 第三に、実験結果を通して、MAE の性能がどの仮説を支持しているのか。  

この視点をもとに、本記事は **MAE は何をうまく切り分けたのか** を読み解いていく。
ではまず、そもそもなぜ画像では masked modeling が NLP ほど素直に効かなかったのか、という点から見ていく。

## 1. なぜ画像では masked modeling がそのまま効かなかったのか

masked modeling の発想自体は単純である。入力の一部を隠し、その欠損部分を予測するようにモデルを学習させる。自然言語処理では、この枠組みはきわめて大きな成功を収めた。BERT に代表される masked language modeling では、文中の一部の token を隠して予測させるだけで、モデルは文法、意味、文脈を考慮せざるを得ない。欠損 token の予測が、そのまま高次の言語理解を要求する課題になっていたのである。

MAE 論文の出発点も、この NLP における成功である。ただし、ここで重要なのは、MAE が BERT の仕組みを画像へ単純移植したわけではないという点である。むしろ論文が最初に問題にしているのは、**言語と画像では信号の性質が根本的に異なる**という事実である。

言語は、人間が離散的な記号列として構成した信号である。各 token は比較的高い意味密度を持ち、ひとつの単語が文全体の意味や構文上の役割を大きく左右する。したがって、少数の token を隠すだけでも、その予測には周辺文脈の理解が必要になりやすい。

これに対して、画像は自然信号である。隣接する pixel や patch は強く相関しており、局所的な連続性やテクスチャの冗長性を多く含んでいる。そのため、画像の一部を隠しても、周囲のエッジ、色、テクスチャをなぞるだけで、それらしい復元ができてしまう場合がある。つまり、画像における単純な masked prediction は、必ずしも物体理解やシーン理解を要求しない。低レベルな補間だけで損失を下げられる余地が大きいのである。

この点が、vision における autoencoding 系手法が NLP ほど素直には伸びなかった理由と対応している。入力を破損させ、それを復元するという枠組み自体は MAE 以前から存在していた。denoising autoencoder も、画像 inpainting も、広い意味では欠損復元を利用した学習である。しかし、それらがそのまま強い視覚表現学習の標準形にならなかったのは、画像再構成がしばしば易しすぎたからである。タスクが易しければ、モデルは大域的な構造を理解する前に、局所統計の延長だけで目的関数を最小化できてしまう。

したがって、vision において masked modeling を成立させるには、単に「隠して復元する」だけでは不十分である。必要なのは、再構成課題を表現学習として意味のある難しさに変えることである。MAE が採用した第一の答えが、75% 前後という高いマスク率である。

低いマスク率では、モデルは多くの周辺情報を見ながら欠損部分を補えばよい。しかし、75% 程度の patch が失われると、状況は大きく変わる。可視 patch は全体の一部に限られ、局所的な連続性だけでは復元できない。モデルは、少数の手掛かりから、物体やシーンの全体構造を推論する必要がある。ここで高マスク率は、単なる強いノイズ注入ではない。 **pretext task を局所補間から構造推論へ押し上げるための設計変数** として機能している。

ただし、タスクを難しくするだけでは十分ではない。画像再構成は本質的に低レベルな出力を要求するため、encoder に再構成の細部まで背負わせると、認識に有用な抽象表現の学習と、画素復元のための局所的 detail の保持が混ざりやすい。そこで MAE は、encoder と decoder の役割を明確に分ける。

MAE の encoder は、マスクされていない visible patch のみを入力として受け取る。mask token は encoder には入れない。encoder は、限られた可視情報から潜在表現を作ることに集中する。一方で、欠損位置を表す mask token は encoder の後に追加され、軽量な decoder が full token 列を受け取って再構成を担う。つまり、encoder は表現学習の中核を担当し、decoder は pretraining 中の再構成タスクを成立させる補助モジュールとして働く。

この非対称設計には、もう一つ重要な意味がある。それは計算効率である。高マスク率を採ると、visible patch は全体の 25% 程度にまで減る。encoder が visible patch のみを処理するなら、重い Transformer encoder が扱う token 数は大幅に減る。self-attention の計算量は系列長に対しておおむね二次的に増えるため、visible-only encoder は学習コストを大きく下げる。

ここが MAE の設計として特に洗練されている点である。高マスク率は、表現学習の観点ではタスクを難しくする。一方で、visible-only encoder と組み合わせると、計算量の観点では学習を軽くする。つまり MAE は、 **表現学習に必要な難しさ** と、 **大規模 ViT を訓練するための軽さ** を、同じ仕組みから同時に引き出している。

このように見ると、MAE の出発点は「画像でも BERT をやる」という単純な模倣ではない。むしろ逆である。MAE は、画像が言語とは異なる信号であることを前提に、vision に適した masked modeling を再設計した手法である。高マスク率、visible-only encoder、lightweight decoder、masked patch のみに対する再構成損失といった後続の設計は、すべてこの問題意識から導かれている。

したがって MAE を理解するとは、Figure 1 の処理フローを単に追うことではない。重要なのは、なぜそのフローでなければならなかったのかを理解することである。MAE は、画像を隠して復元するだけの autoencoder ではない。画像において masked modeling を表現学習として成立させるには、何を隠し、どこで処理し、どこに再構成を任せるべきかを設計した方法なのである。

## 2. MAE の方法：高マスク率・非対称設計・再構成損失

MAE の方法は、次の一文に集約できる。 **入力画像の大部分をマスクし、見えている patch だけを重い encoder に通し、欠損 patch の再構成だけを軽量 decoder に任せる** 。この構成は、単なる実装上の工夫ではない。第1章で見たように、画像では単純な再構成課題が局所補間に流れやすい。したがって、MAE では高いマスク率によって課題を難しくしつつ、visible patch のみを encoder に入力することで計算量を抑える。さらに、再構成の細部は decoder に分担させることで、encoder を表現学習の中心として保つ。

以下では、MAE の処理を、masking、visible-only encoder、lightweight decoder、masked-only reconstruction loss、実装上の工夫の順に整理する。

### 2.1 入力画像の patch 化とランダムマスキング

まず、入力画像を $N$ 個の重なりのない patch に分割する。ViT と同様に、各 patch は線形射影によって token に変換され、positional embedding によって位置情報が加えられる。

入力 patch の添字集合を

$$
\Omega = \{1, 2, \dots, N\}
$$

とする。MAE では、この集合を visible patch の集合 $V$ と masked patch の集合 $M$ に分ける。

$$
V \cup M = \Omega, \qquad V \cap M = \emptyset
$$

mask ratio を $r \in [0,1]$ とすれば、概念的には

$$
|M| \simeq rN, \qquad |V| \simeq (1-r)N
$$

である。MAE の特徴は、この $r$ を非常に大きく取り、標準設定では $r=0.75$ とする点にある。つまり、全 patch のおよそ 75% をマスクし、残り 25% 程度の patch だけを encoder に与える。

ここで重要なのは、masking が block-wise ではなく、基本的には一様ランダムに行われることである。連続した大きな領域を隠すのではなく、画像全体から sparse に patch を残す。これにより、モデルは特定領域の局所補間ではなく、画像全体に散らばった少数の手掛かりから構造を推論することになる。

### 2.2 Encoder は visible patch のみを処理する

MAE の最も重要な設計は、encoder が visible patch のみを入力として受け取る点である。シンプルな masked modeling では、欠損位置を表す mask token を full token 列に挿入し、その全体を encoder に通したくなる。しかし MAE はそうしない。masked patch は encoder 入力から完全に除去され、encoder は $V$ に属する token だけを処理する。

したがって、encoder が扱う系列長は $N$ ではなく $|V|$ である。mask ratio が $r$ のとき、

$$
|V| \simeq (1-r)N
$$

となる。Transformer の self-attention は系列長に対しておおむね二次の計算量を持つため、attention 部分の計算量は概念的に

$$
O(N^2)
$$

から

$$
O(|V|^2)=
O\left(((1-r)N)^2\right)
$$

へ減少する。たとえば $r=0.75$ なら、visible patch 数は $0.25N$ である。このとき attention の二次項だけを見れば、

$$
(0.25)^2 = \frac{1}{16}
$$

となり、full token を処理する場合に比べて大幅に軽くなる。実際の FLOPs には MLP や線形射影なども含まれるため、全体の高速化率が単純に 16 倍になるわけではない。しかし、重い encoder の入力 token 数を削ることが、MAE の計算効率に大きく寄与していることは明確である。

この設計には、計算効率以外の意味もある。encoder は、pretraining 時にも downstream 時にも、実際の画像 patch だけを処理する。もし encoder に大量の mask token を入れて事前学習すると、pretraining 時には人工的な mask token を含む入力を見ているのに、downstream 時にはそれが存在しないという分布ギャップが生じる。MAE は mask token を encoder から排除することで、このギャップを避けている。

### 2.3 Decoder は mask token を含む full token 列を再構成する

encoder の出力は、visible patch に対応する latent token のみである。しかし最終的には、元画像全体の patch を再構成する必要がある。そこで MAE は、encoder の後段で初めて mask token を導入する。

具体的には、encoder から得られた visible token と、masked patch の位置に対応する学習可能な mask token を結合する。その後、それらを元の patch 順序へ並べ戻し、decoder に入力する。decoder 側では、full token 列に対して positional embedding を加える。これにより decoder は、どの位置が visible patch 由来で、どの位置が mask token 由来なのかを、位置情報とともに受け取る。

ここでの decoder は、encoder と同等の重いモデルである必要はない。むしろ MAE では、decoder は encoder よりも浅く狭い lightweight な Transformer として設計される。理由は明確である。decoder の役割は、下流タスクで使う汎用表現を作ることではない。あくまで、pretraining 中に masked patch の再構成タスクを成立させることである。

事前学習が終わると decoder は破棄され、下流タスクでは encoder のみが使われる。したがって MAE では、重い計算資源を encoder に集中させ、decoder は再構成のための補助モジュールとして最小限に保つ。この非対称性が、MAE の方法上の中心である。

### 2.4 損失は masked patch のみに課す

MAE は、decoder の出力から各 patch の画素値を再構成する。patch $i$ の真値を $x_i$、decoder による予測を $\hat{x}_i$ とする。このとき、MAE の基本的な再構成損失は、masked patch 上の平均二乗誤差である。

$$
\mathcal{L}_{\mathrm{MAE}}=
\frac{1}{|M|}
\sum_{i \in M}
\left\lVert \hat{x}_i - x_i \right\rVert_2^2
$$

重要なのは、損失を visible patch ではなく、masked patch のみに対して計算する点である。visible patch は入力として既に与えられているため、それを再構成させても、欠損部分を推論する課題としての意味は弱い。損失を masked patch に限定することで、モデルは「見えていない部分をどう推論するか」に集中する。

これは BERT の masked language modeling と対応する考え方である。ただし、BERT が離散 token の分類問題として欠損語を予測するのに対し、MAE は画像 patch の pixel value を回帰する。すなわち、MAE は離散 token prediction ではなく、pixel reconstruction を通じて masked modeling を実現している。

論文では、再構成 target として生の pixel だけでなく、patch ごとに正規化した pixel も検討されている。各 patch 内で平均と標準偏差を用いて正規化した target を用いると、実験上は表現品質が改善する。これは、MAE においては複雑な tokenizer を別途用意しなくても、pixel-based な再構成目標だけで十分に強い表現学習が可能であることを示している。

### 2.5 実装は shuffle と unshuffle で簡潔に書ける

MAE の処理は、高マスク率、部分観測、非対称 encoder-decoder という構成を持つため、一見すると特殊な sparse attention や複雑なマスク処理が必要に見える。しかし実際の実装は比較的単純である。

基本的な流れは次のように整理できる。

1. 画像を patch 化し、token 列を作る。
2. token の順序をランダムに shuffle する。
3. 先頭の $(1-r)N$ 個を visible token として取り出す。
4. visible token のみを encoder に通す。
5. encoder 出力に mask token を追加する。
6. unshuffle によって元の patch 順序へ戻す。
7. full token 列を decoder に入力し、元画像を再構成する。
8. masked patch のみに対して再構成損失を計算する。

このように、MAE の masking は「full token 列の中に mask token を埋め込んで encoder に通す」操作ではない。むしろ、token 列を shuffle し、visible token だけを切り出して encoder に渡す操作として実装できる。mask token は encoder の後で追加され、decoder に入る直前に元の順序へ戻される。

この実装方針により、MAE は特殊な疎演算を必要としない。標準的な Transformer encoder-decoder 構造と、shuffle / unshuffle のインデックス操作だけで実現できる。方法としての単純さは、MAE が大規模 ViT の事前学習に適している理由の一つである。

以上をまとめると、MAE の方法は次のように整理できる。高マスク率によって再構成課題を難しくする。visible-only encoder によって重い計算を可視 patch に限定する。lightweight decoder によって再構成の細部を引き受ける。masked-only reconstruction loss によって学習の焦点を欠損部分の推論へ絞る。そして、これらを特殊な sparse operation なしに、shuffle と unshuffle によって実装する。

つまり MAE は、複雑な補助損失や特殊な target engineering に頼った方法ではない。高マスク率、非対称 encoder-decoder、masked-only loss という少数の設計を整合的に組み合わせることで、 **難しい pretext task、軽い事前学習、単純な実装** を同時に成立させた方法である。

## 3. MAE は何をうまく切り分けたのか

第2章では、MAE の処理を masking、visible-only encoder、lightweight decoder、masked-only reconstruction loss という要素に分けて整理した。ここで重要なのは、それらを個別の工夫として並べるだけでは、MAE の本質を捉えきれないという点である。

MAE の強さは、単に「画像 patch を隠して復元した」ことにあるのではない。むしろ本質は、画像における masked modeling で衝突しやすい複数の要請を、設計上きれいに切り分けた点にある。

具体的には、MAE は次のような対立を同時に扱っている。

- 再構成課題は難しくしたいが、学習計算量は増やしたくない。
- encoder には認識に有用な表現を学ばせたいが、再構成には画素レベルの detail も必要である。
- masked prediction をしたいが、downstream で使わない mask token を encoder に見せたくない。
- reconstruction loss を使いたいが、最終目的は画像復元ではなく表現学習である。

MAE の設計は、これらの対立を一つずつ整理している。高マスク率は task の難度を上げる。visible-only encoder は重い計算を可視 patch に限定する。lightweight decoder は再構成の detail を引き受ける。mask token の後置は encoder の入力分布を保つ。masked-only loss は学習の焦点を欠損部分の推論に絞る。

この章では、MAE が何をどのように切り分けたのかを、設計原理として整理する。

### 3.1 高マスク率は「局所補間」を「構造推論」に変える

MAE の第一の切り分けは、画像の冗長性をどう扱うかにある。

画像は空間的冗長性が高い。隣接する patch は互いに強く相関し、エッジ、色、テクスチャは局所的に連続しやすい。そのため、低いマスク率で一部の patch だけを隠しても、モデルは周辺 patch からの補間によってそれらしい復元を行えてしまう。この場合、再構成損失は下がるかもしれないが、それが大域的な物体理解やシーン理解につながるとは限らない。

MAE は、この問題に対して高マスク率を用いる。典型的には 75% 程度の patch をマスクし、全体の 25% 程度しか encoder に見せない。これにより、欠損部分は単なる近傍補間では復元しにくくなる。可視 patch が sparse にしか存在しないため、モデルは画像全体の構造を推論しなければならない。

ここで高マスク率は、単に入力を強く壊すためのノイズではない。役割は、pretext task の性質を変えることである。低マスク率では、課題は「周囲を見て空白を埋める」問題になりやすい。一方、高マスク率では、「少数の手掛かりから全体像を推論する」問題になる。

つまり MAE は、画像の冗長性をそのまま放置するのではなく、むしろ逆手に取っている。画像が冗長だからこそ、大部分を隠しても意味的な推論課題を作れる。そして大部分を隠さなければ、再構成課題は表現学習として十分に強くならない。この意味で、高マスク率は MAE の単なるハイパーパラメータではなく、masked modeling を vision で成立させるための中心的な設計変数である。

### 3.2 encoder と decoder の責務を分離した

第二の切り分けは、encoder と decoder の役割分担である。

autoencoder という名前からは、入力を encoder で圧縮し、decoder で元に戻す対称的な構造を想像しやすい。しかし MAE は、通常の autoencoder とはかなり異なる設計思想を持つ。MAE における encoder と decoder は、同じ重さ・同じ責務を持つ対称な部品ではない。

encoder の役割は、少数の visible patch から、下流タスクに使える潜在表現を作ることである。一方、decoder の役割は、pretraining 中に masked patch の再構成を成立させることである。つまり MAE は、 **表現学習の中核** と **再構成タスクの実行部** を明確に分けている。

この分離は重要である。画像再構成は、どうしても低レベルな情報を要求する。色、エッジ、テクスチャ、局所的な detail を復元する必要があるからである。もし重い encoder に再構成の細部まで強く背負わせると、encoder は認識に有用な抽象表現だけでなく、画素復元のための局所情報まで保持する方向に引っ張られる。その結果、表現学習と画素再構成の目的が混ざりやすい。

MAE は、この低レベルな再構成負荷を decoder 側に逃がす。encoder は visible patch から抽象的な特徴を作ることに集中し、decoder は mask token と encoded token を受け取って pixel reconstruction を行う。しかも decoder は事前学習後に破棄される。下流タスクで使われるのは encoder だけである。

したがって、MAE の非対称設計は「decoder を小さくして速くした」というだけの話ではない。より本質的には、encoder に何を学ばせ、decoder に何を押し込めるかを整理した設計である。再構成を pretext task として利用しながらも、encoder を再構成専用器にしない。この責務分離が、MAE の表現学習を支えている。

### 3.3 mask token を encoder に入れない

第三の切り分けは patch と mask token の扱いである。

単純に考えると、masked modeling では、欠損位置を表す mask token を最初から token 列に挿入し、その full token 列を encoder に通す設計が自然に見える。実際、BERT 型の発想に引き寄せるなら、mask token も encoder 入力の一部として扱いたくなる。

しかし MAE はそうしない。mask token は encoder には入れず、encoder の後で decoder 側に追加する。encoder が処理するのは、実際に観測された visible patch だけである。

この設計には二つの意味がある。

第一に、encoder の入力分布を downstream に近づけられる。事前学習中に encoder が大量の mask token を見ていると、downstream 時には存在しない人工 token を前提に表現を作ることになる。これは、pretraining と downstream の間に入力分布のギャップを生む。MAE は、encoder から mask token を排除することで、このギャップを避けている。

第二に、計算量を減らせる。mask token を encoder に入れなければ、encoder の系列長は full patch 数 $N$ ではなく、visible patch 数 $|V|$ になる。高マスク率では $|V|$ が大きく減るため、重い Transformer encoder の計算量も大幅に下がる。

つまり、mask token を encoder に入れないという選択は、表現品質と計算効率の両方に効いている。encoder は実 patch のみを処理するため、downstream で使いやすい表現を学びやすい。同時に、処理 token 数が減るため、大規模 ViT の事前学習が現実的になる。

MAE らしさは、このように一つの設計が複数の問題を同時に解いている点にある。

### 3.4 「難しさ」と「軽さ」を同じ仕組みで両立した

MAE の設計で特に洗練されているのは、pretext task の難化と計算効率の向上が同じ方向を向いていることである。

一般には、学習課題を難しくすると、計算は重くなりやすい。より多くの情報を処理し、より複雑な予測を行い、より大きなモデルが必要になるからである。しかし MAE では逆のことが起きる。高マスク率によって再構成課題を難しくすると、同時に visible patch 数は減る。visible patch 数が減れば、encoder が処理する token 数も減る。したがって、課題を難しくする操作が、そのまま encoder の計算を軽くする操作にもなっている。

これは偶然ではない。MAE は、画像の冗長性を前提にしている。画像には多くの局所相関があるため、低マスク率では課題が易しすぎる。一方で、高マスク率にしても、残った少数の patch から意味的な構造を推論する余地はある。つまり、画像の冗長性は、pretext task を難しくするための余白でもあり、計算量を削るための余白でもある。

MAE はこの性質を利用している。高マスク率によって局所補間を封じ、visible-only encoder によって重い計算を削る。結果として、 **表現学習としての難しさ** と、 **大規模学習としての軽さ** が同時に成立する。

ここが、MAE を単なる masked autoencoder 以上の方法にしている。高マスク率と visible-only encoder は別々の工夫ではない。両者は、画像の冗長性を「課題設計」と「計算効率」の両方に変換するための一体の設計である。

### 3.5 reconstruction は目的ではなく手段である

最後に切り分けるべきなのは、reconstruction と representation learning の関係である。

MAE は訓練時に画像 patch を再構成する。損失も pixel reconstruction の平均二乗誤差で定義される。そのため、表面的には「画像をきれいに復元するモデル」に見えるかもしれない。しかし、MAE の目的は画像復元そのものではない。目的は、再構成課題を通じて、下流タスクに有用な視覚表現を encoder に学習させることである。

この点は、MAE の設計全体から明らかである。まず、損失は masked patch のみに課される。visible patch を含めて画像全体を忠実に復元することが目的なら、すべての patch に損失を課す設計もあり得る。しかし MAE はそうせず、「見えていない部分を推論する」ことに学習信号を絞っている。

また、decoder は事前学習後に破棄される。もし画像復元そのものが目的であれば、decoder は最終成果物の一部である。しかし MAE では、下流タスクで使うのは encoder だけである。これは、decoder が再構成課題を成立させるための訓練用モジュールであり、最終的に欲しいものは encoder の表現であることを示している。

したがって、MAE を理解するうえでは、reconstruction loss を「最終目的」としてではなく、「良い表現を得るための訓練信号」として読む必要がある。MAE の価値は、再構成を行ったことではなく、再構成を表現学習にとって意味のある課題へ変換したことにある。

この観点に立つと、MAE の設計は一貫している。高マスク率は、再構成を局所補間ではなく構造推論に変える。visible-only encoder は、表現学習の中核を実 patch の処理に集中させる。lightweight decoder は、画素復元に必要な detail を引き受ける。masked-only loss は、欠損部分の推論に学習を集中させる。そして decoder は最終的に捨てられ、encoder だけが下流タスクへ渡される。

以上をまとめると、MAE は新しい層や複雑な損失を足した方法ではない。むしろ、画像における masked modeling の失敗しやすい点を整理し、それぞれに対して最小限で整合的な設計を当てた方法である。

高マスク率は task を難しくする。visible-only encoder は計算を軽くし、downstream との分布ギャップを抑える。lightweight decoder は再構成の detail を引き受ける。mask token の後置は encoder を実 patch の処理に集中させる。masked-only reconstruction loss は、学習信号を欠損部分の推論に限定する。

このように、MAE の本質は「masked reconstruction をしたこと」ではない。 **reconstruction を表現学習のための良い pretext task に変換し、そのための役割分担をアーキテクチャ上で明確にしたこと** である。これが、MAE がうまく切り分けたものである。

## 4. 実験から見る MAE の本質

ここまで、MAE の設計を「高マスク率」「visible-only encoder」「lightweight decoder」「masked-only reconstruction loss」という観点から整理してきた。では、それらの設計は実験的にどのように裏づけられているのか。

本章で見るべきなのは、単に「どの設定で何%だったか」ではない。MAE 論文のアブレーションは、それぞれが明確な設計仮説に対応している。したがって、実験結果はスコア表として読むよりも、次の問いに答えるものとして読む方がよい。

- 高マスク率は本当に必要なのか。
- decoder はどの程度の能力を持つべきなのか。
- mask token を encoder に入れない設計は妥当なのか。
- pixel reconstruction は target として十分なのか。
- MAE は強い augmentation に依存しているのか。
- random masking は他の masking strategy より適切なのか。
- MAE は本当に大規模化に向いているのか。

これらの問いを順に見ることで、MAE の強さが個々の工夫の寄せ集めではなく、一貫した設計思想から生じていることが分かる。

### 4.1 高マスク率は本当に必要なのか

MAE の中核仮説は、画像では masked prediction を十分に難しくするために高いマスク率が必要である、というものである。この点を直接検証しているのが **Figure 2** である。

Figure 2 は、masking ratio を横軸に取り、fine-tuning と linear probing の精度がどのように変化するかを示している。結果はかなり明確である。MAE では、**75% 前後という高いマスク率が最もよく機能する**。これは、BERT における典型的な 15% 前後のマスク率とは大きく異なる。また、従来の vision 系 masked modeling で使われることの多かった 20〜50% 程度のマスク率とも異なる。

![](https://static.zenn.studio/user-upload/896397fdc03c-20260423.png)
*Figure 2. **Masking ratio.** 高い masking ratio、特に 75% 前後の設定が、fine-tuning と linear probing の両方で良好に機能する。y 軸は ImageNet-1K validation accuracy を表す。*

この結果が意味しているのは、高マスク率が単なる強い破損ではないということである。低いマスク率では、モデルは周囲の patch を見ながら局所的に補間するだけで損失を下げられる。これに対して、75% 程度の patch が失われると、可視情報はかなり sparse になる。モデルは、少数の手掛かりから画像全体の構造を推論しなければならない。

この解釈は、**Figure 3** の再構成例とも対応している。Figure 3 では、75% の masking ratio で事前学習した MAE に対し、推論時にはさらに高いマスク率の入力を与えたときの再構成が示されている。出力は元画像と完全には一致しない。しかし、意味的にはもっともらしい画像として再構成されている。

![](https://static.zenn.studio/user-upload/254212894aea-20260423.png)
*Figure 3. 75% の masking ratio で事前学習された MAE を、より高い masking ratio の入力に適用した場合の再構成例。出力は元画像と完全一致するわけではないが、意味的には自然な再構成になっている。*

ここで重要なのは、再構成画像の見た目そのものではない。Figure 3 が示しているのは、MAE が単に近傍 patch をなぞっているのではなく、少ない可視情報から画像としてあり得る構造を推論しているという点である。したがって、Figure 2 と Figure 3 は合わせて、**高マスク率が pretext task を局所補間から構造推論へ変えている**ことを支持している。

### 4.2 decoder は重いほどよいのか

次に問うべきなのは、decoder の役割である。MAE では encoder と decoder が非対称に設計される。encoder は大きく、decoder は相対的に軽い。では、decoder はどの程度の能力を持つべきなのか。

この点を検証しているのが **Table 1a** と **Table 1b** である。Table 1a は decoder depth、Table 1b は decoder width に関するアブレーションである。結果を見ると、decoder の能力は特に linear probing に影響する。論文では、decoder depth の調整によって linear probing が大きく改善することが示されている。一方で、fine-tuning に対する影響は比較的小さい。

![](https://static.zenn.studio/user-upload/dec48d261069-20260423.png)
*Table 1. **MAE のアブレーション実験。** ImageNet-1K 上で ViT-L/16 を用いた結果。fine-tuning と linear probing の精度を報告している。デフォルト設定は、decoder depth 8、decoder width 512、reconstruction target は非正規化 pixel、data augmentation は random resized cropping、masking ratio は 75%、pretraining length は 800 epochs である。*

この結果は、decoder が単なる飾りではないことを示している。decoder が弱すぎると、再構成に必要な処理を十分に引き受けられず、encoder 側の表現にも悪影響が出る。特に linear probing では、encoder 表現の性質が直接反映されやすいため、decoder 設計の影響が大きく見える。

ただし、ここから「decoder は大きいほどよい」と結論してはいけない。MAE の decoder は、あくまで pretraining 中の再構成を成立させるための補助モジュールである。事前学習後には破棄され、下流タスクでは encoder だけが使われる。したがって、decoder は十分な再構成能力を持つ必要はあるが、主役ではない。

この実験が支持しているのは、次のような解釈である。MAE では、再構成に必要な低レベル処理の一部を decoder 側へ逃がすことで、encoder をより表現学習に集中させている。decoder がある程度の能力を持つことは重要だが、最終的に価値を持つのは encoder の表現である。つまり、Table 1a と Table 1b は、**encoder と decoder の責務分離が有効である**ことを示している。

### 4.3 mask token を encoder に入れない設計は妥当なのか

MAE のもう一つの中心設計は、mask token を encoder に入れないことである。mask token は encoder の後で追加され、decoder にのみ入力される。この設計は、性能と計算効率の両方に効いている。

この点を直接検証しているのが **Table 1c** である。Table 1c では、mask token を encoder に入れる場合と、MAE のように encoder から除外する場合が比較されている。結果として、mask token を encoder に入れると linear probing の性能が大きく低下する。これは、単なる小さな差ではなく、設計方針そのものを左右する差である。

この低下は、pretraining と downstream の入力分布ギャップとして理解できる。encoder が事前学習中に大量の mask token を含む入力を見ていると、下流タスク時には存在しない人工 token に依存した表現を作る可能性がある。一方、MAE の encoder は pretraining 時にも downstream 時にも、実際の画像 patch だけを処理する。この一貫性が、表現品質にとって重要である。

さらに、この設計は計算効率にも直結している。mask token を encoder に入れない場合、encoder が処理する token 数は full patch 数 $N$ ではなく、visible patch 数 $|V|$ になる。高マスク率では $|V|$ が大きく減るため、重い Transformer encoder の計算量も削減される。

この計算効率の効果を示しているのが **Table 2** である。Table 2 では、mask token を encoder に含める設定を基準として、MAE の visible-only encoder が実時間で大きく高速化することが示されている。

![](https://static.zenn.studio/user-upload/a5aaf009a1dd-20260423.png)
*Table 2. MAE pretraining の実時間比較。128 個の TPU-v3 core と TensorFlow を用いた 800 epochs の benchmark。speedup は、encoder が mask token を持つ設定に対する相対値である。decoder width は 512、masking ratio は 75% である。*

したがって、mask token を encoder に入れない設計は、単なる高速化テクニックではない。表現品質の観点では downstream との分布ギャップを抑え、計算効率の観点では重い encoder の token 数を削減する。Table 1c と Table 2 は、この一つの設計が性能と効率の両方に効いていることを示している。

### 4.4 pixel reconstruction は target として十分なのか

masked image modeling では、何を予測対象にするかも重要な論点である。pixel を直接再構成するのは簡単すぎるのではないか。離散 token や別の特徴量を target にした方がよいのではないか。この疑問に対応するのが **Table 1d** である。

Table 1d では、非正規化 pixel、patch ごとに正規化した pixel、PCA、dVAE token などが比較されている。結果として、正規化 pixel target は良好に機能する。一方で、PCA target は性能を落とす。また、dVAE token は一部の設定で非正規化 pixel よりよいが、正規化 pixel に対して明確な優位を示すわけではない。

この結果は、MAE の強さが複雑な target engineering に依存していないことを示している。特別な tokenizer を事前学習しなくても、pixel-based な再構成目標で十分に強い表現学習が成立する。もちろん、生の pixel をそのまま使うよりも、patch 内で正規化した pixel target の方が有効である。これは、局所的な輝度やコントラストの差をある程度取り除き、形状や構造に関係する情報を学びやすくするためだと考えられる。

この論点は、転移学習を含む比較でも確認される。**Table 3** では、pixels と tokens を再構成 target として比較しており、正規化 pixel target が dVAE token に対して統計的に劣らないことが示されている。

![](https://static.zenn.studio/user-upload/dbcb44d7b6b6-20260423.png)
*Table 3. **MAE の reconstruction target としての pixels と tokens の比較。** 正規化 pixel target と dVAE token target の差は統計的に有意ではない。*

したがって、Table 1d と Table 3 から読み取るべきことは明確である。MAE の本質は、複雑な token prediction にあるのではない。むしろ、高マスク率と非対称設計によって pretext task を適切に作れば、pixel reconstruction という単純な目標でも十分に強い表現学習が可能になる。

### 4.5 MAE は強い augmentation に依存しているのか

contrastive learning に慣れていると、自己教師あり学習では強い data augmentation が不可欠であるように感じられる。SimCLR や MoCo 系の手法では、augmentation は単なる前処理ではなく、「どの不変性を学ばせるか」を決める中心的な設計要素である。

しかし、MAE の挙動はそれとは異なる。この点を示しているのが **Table 1e** である。Table 1e では、cropping、flipping、color jittering などの augmentation が比較されている。結果として、MAE は比較的弱い augmentation でも良好に機能する。むしろ、color jittering のような強い augmentation は性能を悪化させる場合がある。

この結果は、MAE における多様性の源泉が augmentation ではなく、random masking にあることを示している。MAE では、同じ画像でも iteration ごとに異なる patch が可視になり、異なる patch が隠される。したがって、モデルは毎回異なる部分観測から欠損領域を推論することになる。

つまり MAE は、contrastive learning のように「二つの augmented view を作り、それらを近づける」方法ではない。代わりに、「同じ画像から異なる sparse observation を作り、欠損部分を推論させる」方法である。このため、強い augmentation によって無理に view の多様性を作らなくても、random masking 自体が十分な学習信号を生み出す。

Table 1e は、MAE が augmentation に依存しないというより、**augmentation とは別の仕方で自己教師あり学習の多様性を確保している**ことを示している。

### 4.6 random masking は本当に適切なのか

masking ratio だけでなく、どのように patch を隠すかも重要である。この点を検証しているのが **Table 1f** と **Figure 4** である。

![](https://static.zenn.studio/user-upload/7cf343ccb993-20260423.png)
*Figure 4. masking strategy の比較。左は random sampling、中央は block-wise sampling、右は grid-wise sampling を示す。masking strategy は pretext task の難易度と再構成の性質を大きく変える。*

grid-wise sampling では、patch が規則的に残るため、再構成は比較的なめらかで見た目もよくなりやすい。しかし、表現学習としては必ずしも強くない。これは、規則的な観測構造では補間課題が易しくなりすぎるからである。

一方、block-wise sampling では、大きな領域をまとめて隠すため、局所的には難しい課題になる。しかし、高マスク率では可視情報の分布が偏りやすく、学習が不安定になりやすい。論文では、block-wise masking は 50% ではある程度機能するものの、75% では性能が悪化することが示されている。

これに対して、random sampling は画像全体に sparse な可視 patch を残す。局所補間だけでは解きにくく、かつ画像全体から手掛かりを得られる。したがって、高い masking ratio と組み合わせたときに、task difficulty と学習可能性のバランスがよい。

この結果から分かるのは、再構成が見た目にきれいであることと、表現学習に有効であることは別だということである。MAE が目指しているのは、最も自然な画像補完ではない。下流タスクに有用な encoder 表現を得ることである。その意味で、random masking は、見た目の再構成品質よりも representation learning の観点から選ばれた strategy である。

### 4.7 MAE は本当にスケールするのか

最後に、論文タイトルにも含まれる “Scalable” が何を意味するのかを見る。MAE のスケーラビリティは、単に大きなモデルで動いたという意味ではない。長く学習できること、大きな ViT に適用できること、そして計算時間の面でも現実的であることを含んでいる。

## 5. MAE の表現は「線形には弱く、適応すると強い」

ここまでの議論から、MAE が高マスク率と非対称設計によって強い視覚表現を学習していることは分かった。しかし、ここで一つ注意すべき点がある。MAE の強さは、linear probing だけを見ると過小評価されやすい。

自己教師あり学習では、学習済み encoder を凍結し、その上に線形分類器だけを載せて評価する linear probing がよく使われる。この評価は、表現がどれだけ線形分離しやすいかを見るうえでは便利である。しかし、表現の有用性をすべて測れるわけではない。特に MAE のように、reconstruction を通じて豊かな内部表現を学ぶ手法では、線形分類器だけでは情報を十分に引き出せない場合がある。

MAE 論文はこの点を、partial fine-tuning の実験によって示している。重要なのは、MAE の表現が「弱い」のではなく、 **線形に読み出しやすい形で整理されているとは限らない** ということである。少しだけ非線形な適応を許すと、MAE の表現は急速に強さを発揮する。

![](https://static.zenn.studio/user-upload/3e85ba5a22f7-20260423.png)
*Figure 7. **Partial fine-tuning.** Table 1 のデフォルト設定で事前学習した ViT-L に対して、fine-tune する Transformer block 数を変化させた結果。0 blocks は linear probing、24 blocks は full fine-tuning に対応する。MAE 表現は linear probing では MoCo v3 より低いが、1 block 以上を fine-tune すると一貫して MoCo v3 を上回る。*

### 5.1 partial fine-tuning が示すもの

Figure 7 は、MAE 表現に対して「どこまで encoder を更新するか」を変えたときの性能を示している。横軸は fine-tune する Transformer block の数である。0 blocks は encoder を完全に凍結し、線形分類器だけを学習する設定である。つまり linear probing に対応する。一方、24 blocks は ViT-L の全 Transformer block を更新する設定であり、full fine-tuning に対応する。

この図が示しているのは、linear probing から full fine-tuning までを離散的に比較するのではなく、encoder の適応量を連続的に増やしたときに、性能がどのように立ち上がるかである。これにより、MAE の表現が「線形分類器だけでどの程度読めるのか」と、「少し非線形な更新を許すとどこまで伸びるのか」を同時に確認できる。

最も重要なのは、MAE では 1 block だけ fine-tune するだけでも精度が大きく向上する点である。論文では、linear probing の 73.5% から、1 block の fine-tuning によって 81.0% まで上昇すると報告されている。さらに、最後の block のうち MLP sub-block だけを調整した場合でも 79.1% に達する。これは、線形分類器だけを載せる場合よりも大幅に高い。

この結果は、MAE の表現の中に有用な情報が含まれていないというより、その情報が線形分類器だけでは取り出しにくい形で存在していることを示している。ほんの少し encoder 側の非線形変換を許すだけで性能が大きく伸びるためである。

### 5.2 linear probing は表現の一側面しか測らない

この結果から分かるのは、linear probing の値だけで MAE の表現品質を判断するのは危険だということである。

linear probing は、学習済み表現 $z$ に対して線形分類器

$$
\hat{y} = Wz + b
$$

を学習し、その分類精度を見る評価である。ここで $z$ は encoder が出力する特徴、$W$ と $b$ は線形分類器の重みとバイアスである。この評価が高いということは、クラス情報が $z$ 空間上で線形に分離しやすいことを意味する。

しかし、これはあくまで「線形に読み出しやすいか」を測っているにすぎない。表現の中に下流タスクに有用な情報が含まれていても、その情報が線形境界で簡単に分離できるとは限らない。少し非線形な変換を加えれば取り出せる情報も、linear probing では低く評価される。

MAE の場合、まさにこの状況が起きていると考えられる。Figure 7 では、linear probing の時点では MoCo v3 より低い。しかし、1 block 以上を fine-tune すると、MAE は一貫して MoCo v3 を上回る。これは、MAE 表現が線形分離可能性という観点では控えめである一方、非線形に適応させると非常に強く働くことを示している。

論文ではこの性質を、MAE の表現は “linearly less separable but more powerful nonlinear features” である、と表現している。つまり、MAE の表現は線形評価では弱く見えるが、少し適応させると強い非線形特徴として機能する。

> DINO と BEiT に関しての比較がここでないのはなぜだろう...(。´･ω･)?
> たとえ負けていたとしても、迫っているのであれば計算効率の観点から全くマイナスにならないと思うんだが。線形評価では過小評価してしまうことだけを主張することにしたんだろうか。

### 5.3 MoCo v3 との比較から分かること

Figure 7 で重要なのは、MAE が MoCo v3 と比較されている点である。MoCo v3 は contrastive learning 系の強力な自己教師あり手法であり、linear probing では MAE より高い性能を示す。

もし評価を linear probing だけに限定すれば、「MoCo v3 の方が MAE より強い表現を学んでいる」と結論したくなる。しかし Figure 7 は、その判断が不十分であることを示している。少しでも encoder の fine-tuning を許すと、MAE は MoCo v3 を上回る。論文では、4 blocks を fine-tune する設定で MAE が MoCo v3 を 2.6% 上回ると報告されている。

この比較が示しているのは、自己教師あり表現の評価には複数の軸があるということである。contrastive learning 系の手法は、augmentation によって同一画像由来の view を近づけるため、クラスに対応する情報が線形に分離しやすい形で整理されやすい可能性がある。一方、MAE は欠損部分の再構成を通じて、部分観測から全体構造を推論する表現を学ぶ。その結果、線形分類器だけでは読み出しにくいが、少し適応させると強く機能する表現になる。

したがって、MoCo v3 との比較から読み取るべきなのは、「どちらが常に優れているか」ではない。むしろ、pretext task の違いによって、表現の性質が変わるということである。linear probing はその違いの一部しか見ない。MAE の強さを理解するには、partial fine-tuning や full fine-tuning まで含めて評価する必要がある。

### 5.4 なぜ MAE は線形に読み出しにくいのか

では、なぜ MAE の表現は linear probing では低く見えやすいのか。論文はこの点を厳密な理論として展開しているわけではないが、MAE の設計思想から自然に解釈できる。

MAE の pretext task は、少数の visible patch から masked patch を再構成することである。これは単なる分類課題ではない。モデルは、部分的な観測から画像全体の構造を推論し、欠損領域としてもっともらしい pixel を生成する必要がある。そのため encoder が学ぶ表現は、クラス境界に直結する特徴だけではなく、形状、配置、物体の構造、シーンの文脈など、再構成に必要な多様な情報を含みうる。

このような表現は、必ずしも線形分類器でそのまま読み出しやすいとは限らない。情報は含まれていても、クラスごとに単純な線形境界で分離される形に並んでいない可能性がある。つまり、linear probing が低いことは、情報がないことを意味しない。情報の配置が線形読出しに最適化されていない、という可能性がある。

一方、最後の Transformer block や MLP sub-block を少し調整すると、表現空間を下流タスクに合わせて再配置できる。Figure 7 で 1 block fine-tuning によって精度が急激に上がるのは、この再配置がごく少数のパラメータ更新で可能であることを示している。MAE 表現は、線形分類器だけでは硬いが、少し曲げれば下流タスクにうまく合う。そういう性質を持つ表現だと考えられる。

この観点から見ると、MAE の表現は「線形には弱いが、情報量が少ない」のではない。むしろ、情報を豊富に含むが、それを取り出すには少し非線形な読み出しが必要な表現である。これが、論文のいう “more powerful nonlinear features” の意味である。

### 5.5 reconstruction 型 SSL の評価で注意すべきこと

この章の議論は、MAE だけでなく reconstruction 型の自己教師あり学習全般にも関係する。

contrastive learning では、positive pair を近づけ、negative pair を遠ざける、あるいは collapse を避けながら view-invariant な表現を作る。このような目的関数は、下流の分類境界と比較的相性がよい場合がある。そのため linear probing が高くなりやすい。

一方で、MAE のような reconstruction 型 SSL は、分類境界そのものを直接整えるわけではない。入力の一部から全体を推論するための内部表現を作る。これは、分類以外の下流タスク、たとえば検出、セグメンテーション、密な予測、あるいは fine-tuning 前提の転移学習では有利に働きうる。しかし、linear probing だけではその利点が見えにくい。

したがって、MAE のような手法を評価するときには、次の区別が重要になる。

- 線形分類器だけでどれだけ読めるか。
- 少数の block を調整するとどれだけ伸びるか。
- full fine-tuning でどこまで到達するか。
- 分類以外の downstream task へ転移したときにどう振る舞うか。

Figure 7 は、このうち最初の三つを一つの軸上で見せている。0 blocks では MAE は線形に読みにくい。しかし、少数 block の適応で急速に伸びる。そして full fine-tuning では強い性能に到達する。これは、MAE の表現を理解するうえで非常に重要な観察である。

### 5.6 この章のまとめ

第5章の要点は、MAE の表現を linear probing だけで判断してはいけない、ということである。

MAE は、linear probing では contrastive learning 系の手法に劣る場合がある。しかし、それは MAE の表現が弱いことを意味しない。Figure 7 が示すように、1 block だけ fine-tune するだけでも精度は大きく向上し、少数 block の partial fine-tuning では MoCo v3 を上回る。これは、MAE の表現が線形には読み出しにくい一方で、少し非線形に適応させると非常に強く働くことを示している。

この性質は、MAE の pretext task と整合している。MAE は、クラス境界を直接作るのではなく、部分観測から全体構造を推論する表現を学ぶ。そのため、表現は必ずしも線形分離しやすい形にはならない。しかし、下流タスクに合わせて少し調整すれば、豊富な情報を活かせる。

したがって、MAE の表現は「線形には弱く、適応すると強い」と理解するのがよい。第4章のアブレーションが MAE の設計原理を裏づけていたとすれば、第5章の partial fine-tuning は、その設計から生まれる表現の性質を明らかにしている。MAE の強さは、線形評価の一点ではなく、適応後にどれだけ立ち上がるかまで含めて評価すべきである。

## 6. MAE が残したもの

ここまで、MAE の方法、設計原理、実験結果、そして表現の性質を見てきた。では、MAE は最終的に何を残した論文だったのか。

MAE の貢献は、ImageNet で高い精度を達成したことだけではない。もちろん、ViT-L や ViT-H のような大規模 Vision Transformer に対して、自己教師あり事前学習が有効であることを示した点は重要である。しかし、それ以上に大きいのは、 **vision における masked modeling の設計原理を与えたこと** である。

MAE 以前にも、入力を破損させて復元する autoencoding 系の発想は存在していた。画像の一部を隠して復元する inpainting 的な課題もあった。しかし、それらは必ずしも大規模な視覚表現学習の標準形にはならなかった。理由は、これまで述べてきた通りである。画像は局所的な冗長性が高く、単純な再構成課題では、モデルが低レベルな補間だけで損失を下げられてしまうからである。

MAE が重要なのは、この問題に対して「再構成を使うかどうか」ではなく、「再構成をどのように表現学習の課題へ変換するか」を示した点にある。高マスク率によって局所補間を難しくし、visible-only encoder によって重い計算を可視 patch に限定し、lightweight decoder によって画素復元の detail を引き受けさせる。この組み合わせによって、MAE は reconstruction を単なる画像復元ではなく、強い視覚表現を学ぶための pretext task に変えた。

### 6.1 vision SSL に reconstruction という強い路線を戻した

第一の意義は、MAE が vision SSL において reconstruction 型の路線を再び強力な選択肢として示したことである。

MAE が登場する以前、vision の自己教師あり学習では contrastive learning やその周辺手法が大きな成功を収めていた。SimCLR、MoCo、BYOL、DINO などの系譜では、異なる view 間の対応関係を利用し、augmentation によって不変性を学習することが中心的な設計になっていた。そこでは、「どのような view を作るか」「どの不変性を学ばせるか」が性能を左右する重要な問題だった。

これに対して MAE は、異なる view を近づけるのではなく、入力の大部分を隠し、残された情報から欠損部分を推論させる。つまり、contrastive learning が「不変性を作る」方向の自己教師あり学習だとすれば、MAE は「欠損から構造を推論させる」方向の自己教師あり学習である。

この転換は大きい。MAE は、reconstruction 型の SSL が単なる古典的 autoencoder の焼き直しではなく、適切に設計すれば大規模 ViT の事前学習として十分に強いことを示した。これにより、vision SSL の中心的な問いは、「どの augmentation で不変性を学ばせるか」だけではなく、「どのような欠損課題を与えれば、意味ある推論が生じるか」にも広がったのである。

### 6.2 複雑な target engineering に依存しない道を示した

第二の意義は、MAE が複雑な reconstruction target を必須としなかったことである。

masked image modeling では、何を予測対象にするかが大きな論点になる。画像 patch の pixel value を直接予測するのは単純すぎるようにも見える。実際、離散 token を作ってそれを予測する方法や、事前学習済み tokenizer を使う方法も有力な選択肢として存在していた。

しかし MAE は、patch ごとに正規化した pixel target で十分に強い性能を示した。これは、MAE の本質が target の複雑さではなく、task design と architecture design にあることを示している。

もちろん、pixel reconstruction が常に最良の target であるという意味ではない。後続研究では、特徴量 target、token target、マルチモーダル target など、さまざまな方向に発展していく。しかし、MAE が示した重要な点は、少なくとも vision において、複雑な tokenizer を別途用意しなくても、適切な masking と encoder-decoder 設計があれば、pixel target だけで強い自己教師あり学習が成立するということである。

これは方法として非常に大きい。なぜなら、target engineering に依存しすぎる手法は、実装や再現性、スケーリングの面で複雑になりやすいからである。MAE は、単純な pixel reconstruction を使いながらも、設計の置き方次第で強力な表現学習が可能であることを示した。この単純さが、MAE の普及性と拡張性を支えている。

### 6.3 大規模 ViT を自己教師ありで訓練する現実的な道筋を示した

第三の意義は、MAE が大規模 Vision Transformer の自己教師あり事前学習を現実的なものにしたことである。

ViT は強力である一方、自己教師ありで大規模に事前学習するには計算コストが問題になる。特に Transformer の self-attention は token 数に対して二次的に重くなるため、画像 patch をすべて encoder に通す設計では、モデルを大きくするほど負担が大きくなる。

MAE はここで、高マスク率と visible-only encoder を組み合わせた。75% 程度の patch をマスクし、残り 25% 程度の visible patch だけを encoder に通す。これにより、pretext task は難しくなる一方で、encoder の計算量は大きく減る。つまり MAE は、学習課題を強くしながら、同時に計算を軽くするという、通常は両立しにくい二つの要請を同じ仕組みで満たした。

この設計によって、ViT-L や ViT-H のような大規模モデルでも、長い self-supervised pretraining が現実的になる。論文タイトルにある “Scalable” とは、単に大きなモデルで高精度が出たという意味ではない。高マスク率と visible-only encoder によって、 **大規模化したときにも計算効率と表現学習の難しさが両立する** という意味である。

この点は、MAE が後続研究に与えた影響を考えるうえで特に重要である。MAE は、大規模 ViT を自己教師ありで訓練するための実用的な設計パターンを与えた。これは、単なる性能改善ではなく、学習スケールの設計を変えた貢献である。

### 6.4 masked image modeling の基準点を作った

第四の意義は、MAE が masked image modeling を議論するための基準点になったことである。

MAE の構成は非常に明快である。高マスク率、random masking、visible-only encoder、lightweight decoder、masked-only reconstruction loss、pixel target。この組み合わせは、その後の masked image modeling 研究において、一つの標準的な参照点になった。

後続研究は、この枠組みのどこを変えるかによって整理できる。たとえば、masking strategy を変える研究がある。reconstruction target を pixel から feature や token に変える研究がある。decoder を弱くする、あるいはなくす方向もある。画像以外のモダリティ、たとえば video、audio、spectrogram、point cloud、medical image などへ拡張する研究もある。

しかし、どの方向に進むとしても、MAE が与えた設計座標は無視できない。どれくらい隠すのか。encoder は何を見るのか。mask token はどこで入れるのか。decoder にどこまで再構成を任せるのか。損失はどこに課すのか。target は pixel でよいのか、それとも別の表現を使うべきか。これらの問いは、MAE によって明確な形で整理された。

この意味で、MAE は一つの手法であると同時に、masked image modeling の設計空間を定義した論文でもある。後続研究は、MAE をそのまま使う場合も、MAE に反論する場合も、基本的には MAE が置いた座標系の上で議論することになる。

### 6.5 「単純な方法がスケールする」ことを示した

第五の意義は、MAE が「単純な方法がスケールする」という深層学習らしい性質をよく示している点である。

MAE の構成要素は、個別に見ると極端に複雑ではない。画像を patch に分ける。ランダムに多くを隠す。見えている patch だけを encoder に通す。mask token を後で追加する。decoder で pixel を再構成する。masked patch のみに loss を課す。方法の骨格は非常に簡潔である。

それにもかかわらず、MAE は大規模 ViT に適用したときに強い性能を示す。これは、手法の複雑さによって性能を稼いでいるというより、設計原理がスケールと噛み合っていることを意味する。高マスク率によって task difficulty を保ち、visible-only encoder によって計算効率を保ち、decoder を軽くすることで pretraining と downstream の役割を分ける。この構造が大規模化に適していたのである。

深層学習では、しばしば「単純だがスケールする方法」が大きな流れを作る。MAE もその一つとして読める。複雑な補助損失や重い事前処理に依存せず、単純な masked reconstruction を、画像に合う形へ設計し直した。その結果として、大規模モデル、大規模 pretraining、下流タスクへの転移という流れに乗ることができた。

この単純さは、研究上も実装上も重要である。方法が単純であれば、何が効いているのかを検証しやすい。拡張もしやすい。異なるモダリティにも移植しやすい。MAE が広く影響を持った理由の一つは、この設計の単純さにある。

### 6.6 MAE の本当のインパクト

以上を踏まえると、MAE が残したものは「高精度な pretrained model」だけではない。より本質的には、次の三つである。

第一に、画像における masked modeling を成立させるための設計原理を示したこと。  
第二に、高マスク率と visible-only encoder によって、pretext task の難しさと計算効率を同時に成立させたこと。  
第三に、reconstruction 型 SSL を、大規模 ViT の事前学習として現実的で強力な路線に押し上げたこと。

MAE は、画像を隠して復元するという古典的にも見える発想を、Vision Transformer 時代に合わせて再設計した論文である。したがって、MAE のインパクトは、特定のベンチマークスコアだけでは測れない。MAE は、vision SSL において「masked modeling をどう設計すればよいか」という問いに対し、単純で、効率的で、スケールする一つの答えを与えた。その答えが、その後の masked image modeling、自己教師あり ViT、さらに画像以外の信号への masked modeling 研究にまで影響を与えた。

要するに、MAE が残したものは、手法そのもの以上に、設計思想である。画像は言語とは異なる。だからこそ、masked modeling も画像に合わせて作り直す必要がある。MAE はその作り直し方を、非常に単純な形で示した論文である。

## 7. まとめ

ここまで、MAE を単なるアーキテクチャ紹介ではなく、設計思想として読んできた。最後に議論を圧縮すると、MAE の本質は **masked reconstruction そのものではなく、masked reconstruction を vision に適した表現学習課題へ作り直したこと** にある。

MAE の出発点は、NLP における masked language modeling の成功である。しかし、画像は言語とは異なる。言語 token は意味密度が高く、少数の token を隠すだけでも文脈理解を要求しやすい。一方で、画像は空間的冗長性が大きく、単純な欠損補完では局所的な補間だけで解けてしまう場合がある。したがって、vision に masked modeling を持ち込むには、画像という信号に合わせた再設計が必要だった。

MAE は、その再設計を非常に単純な形で実現した。第一に、75% 前後という高マスク率によって、再構成課題を局所補間から構造推論へ押し上げた。第二に、encoder には visible patch のみを入力し、mask token を encoder の後で導入することで、downstream との入力分布ギャップを抑えつつ、重い Transformer の計算量を削減した。第三に、lightweight decoder に再構成の detail を引き受けさせることで、encoder を表現学習の中核として保った。第四に、loss を masked patch のみに課すことで、学習信号を欠損部分の推論に集中させた。

この構成が優れているのは、各要素がばらばらの工夫ではないからである。高マスク率は task を難しくする。同時に、visible patch 数を減らすため encoder の計算を軽くする。decoder は再構成を成立させるが、下流タスクでは破棄される。mask token は再構成には必要だが、encoder には入れない。つまり MAE は、表現学習としての難しさ、計算効率、役割分担、downstream で使う表現の純度を、同じ設計の中で同時に整理している。

実験結果も、この設計思想を一貫して支持していた。高マスク率は fine-tuning と linear probing の両方で有効であり、random masking は task difficulty と学習可能性のバランスを取っていた。decoder 設計は encoder 表現の性質に影響し、mask token を encoder に入れない設計は性能と効率の両方に効いていた。さらに、複雑な tokenizer を使わなくても、正規化 pixel target で十分に強い性能が得られた。これらは、MAE の本質が target の複雑さではなく、task design と architecture design にあることを示している。

一方で、MAE の表現は linear probing だけでは評価しきれない。MAE は線形に読み出しやすい表現というより、少し fine-tuning したときに強く立ち上がる表現を学ぶ。これは、MAE がクラス境界を直接整えるのではなく、部分観測から全体構造を推論するための豊かな内部表現を学ぶことと整合している。したがって、MAE の強さは「線形に分離しやすいか」だけではなく、「downstream に適応させたときにどれだけ機能するか」まで含めて評価する必要がある。

MAE が残した最大の意義は、vision SSL において reconstruction 型の自己教師あり学習を、再び強力でスケール可能な路線として示したことである。contrastive learning が「どの augmentation によって不変性を学ばせるか」を中心課題にしていたのに対し、MAE は「どの欠損課題を与えれば意味ある推論が生じるか」を中心課題にした。これにより、自己教師あり学習の設計空間は大きく広がった。

ただし、MAE を一般化するときには注意が必要である。MAE の設計は画像の性質に深く依存している。別のモダリティへ応用するなら、masking 単位、mask ratio、reconstruction target、decoder の責務、downstream での使い方を再定義しなければならない。MAE の思想は移植できるが、画像で成功した具体設計をそのまま移すだけでは不十分である。

結局、MAE を理解するうえで最も重要なのは、次の一点である。

$$
\boxed{
\begin{aligned}
&\text{MAE は、単に画像を隠して復元する方法ではなく、}\\
&\textbf{「何を隠し、何を見せ、何を再構成させれば、強い表現が生まれるか」}\\
&\text{を設計した方法である。}
\end{aligned}
}
$$

この視点に立つと、MAE の高マスク率、visible-only encoder、lightweight decoder、masked-only loss は、単なる実装上の工夫ではなく、すべて同じ設計原理から導かれていることが分かる。画像は冗長である。だからこそ大きく隠す必要がある。大きく隠すからこそ、encoder の計算は軽くなる。再構成には detail が必要である。だからこそ decoder に任せる。下流で使うのは encoder である。だからこそ encoder には可視 patch のみを見せる。

MAE の美しさは、この整合性にある。単純な方法でありながら、画像の性質、Transformer の計算特性、自己教師あり学習の目的、下流タスクへの転移を同時に見ている。だからこそ MAE は、単なる一手法ではなく、Vision Transformer 時代の自己教師あり学習を考えるうえでの重要な基準点になったのである。
