## はじめに

近年の視覚認識では、ラベル付きデータだけに依存するのではなく、ラベルなしデータから汎用的な表現を獲得する自己教師あり学習 (Self-Supervised Learning; SSL) が重要な位置を占めるようになっている。

なかでも、画像の一部を隠し、残された情報から欠損領域を復元する **masked image modeling (MIM)** は、Vision Transformer (ViT) と Masked Autoencoder (MAE) の成功によって、代表的な事前学習手法の一つとなった。

しかし、ここで一つ問題がある。

ViT でうまく機能した masked image modeling は、ConvNet にもそのまま適用できるのだろうか。

一見すると、答えは単純に思える。画像の一部を mask し、それを復元させればよい。だが実際には、ConvNet と ViT では入力の扱い方も、計算の流れも、特徴表現の作られ方も大きく異なる。ViT では画像を patch token の列として扱えるため、mask された token を encoder から取り除くことができる。一方、ConvNet は dense な二次元 feature map と畳み込み演算を前提としており、同じ設計をそのまま移植することはできない。

ConvNeXt V2 の論文が面白いのは、この問題に対して、単に ConvNet に MAE を載せるのではなく、自己教師あり学習の枠組みとアーキテクチャを **同時に設計** し直している (Co-designing) 点にある。

本記事では、ConvNeXt V2 を単なる高性能な ConvNet としてではなく、**ConvNet で masked image modeling を成立させるための設計論** として読む。具体的には、FCMAE によって ConvNet 向けの masked autoencoder をどのように構成したのか、そして GRN によって自己教師あり学習中の特徴崩壊をどのように抑えたのかを見ていく。

## 1. なぜ ConvNet で自己教師あり学習を考えるのか

### 1.1 ConvNet はまだ舞える

近年の視覚認識では、Vision Transformer (ViT) を中心とする transformer 系アーキテクチャが大きな存在感を持つようになった。とくに、大規模データと大規模モデルを組み合わせたときのスケーリング性能は強力であり、自己教師あり学習との相性もよい。

一方で、ConvNet が過去のものになったわけではない。

ConvNeXt は、従来型の ConvNet を現代的な設計へと更新することで、純粋な畳み込みモデルであっても transformer 系モデルに近い性能を達成できることを示した。depthwise convolution、大きめの kernel、inverted bottleneck 的な構造、Layer Normalization などを取り入れることで、ConvNet は古典的な CNN から大きく変化している。

重要なのは、ConvNet には ViT とは **異なる帰納バイアスがある** という点である。

ConvNet は局所的な空間構造を扱いやすく、画像の平行移動に対して自然な性質を持つ。また、階層的に特徴を抽出するため、低レベルなエッジやテクスチャから、高レベルな意味的特徴へと段階的に表現を作る。この性質は、**画像認識において依然として有効** である。

したがって、ConvNet で自己教師あり学習を成立させることには意味がある。
それは単に「ViT の代替モデルを作る」という話ではない。ConvNet の帰納バイアスを保ったまま、ラベルなしデータから有用な表現を学習できるか、という問題である。

### 1.2 ラベルなしデータから表現を作るための設計

自己教師あり学習は、ラベルなしデータを使ってモデルを事前学習する方法である。ただし、それを「ラベルを使わない学習」とだけ捉えると本質を見失いやすい。

より重要なのは、自己教師あり学習が **学習信号そのものを設計する問題** だという点である。

教師あり学習では、人間が与えたクラスラベルを予測することで表現を学習する。画像分類であれば、モデルは入力画像から「犬」「猫」「車」といったラベルを予測するように訓練される。このとき、何を予測すべきかはデータセット側で定義されている。

一方、自己教師あり学習では、明示的なラベルが存在しない。そのため、モデルに何を予測させるのかを人間が設計する必要がある。

たとえば、contrastive learning では、異なる拡張を施した同一画像を近づけ、別画像を遠ざける。masked image modeling では、画像の一部を隠し、残された情報から欠損領域を復元させる。どちらも、ラベルなしデータから学習信号を作るための pretext task である。

このとき重要なのは、**pretext task がモデルにどのような表現を要求しているか** である。

画像の一部を隠して復元させる場合、モデルは局所的な画素値を単に補間するだけでは不十分である。周囲の文脈、物体の形状、テクスチャ、空間的な連続性などを利用しなければならない。つまり、復元というタスクを通じて、下流タスクにも有用な視覚表現を学習することが期待される。

ただし、ここには落とし穴もある。
pretext task が簡単すぎれば、モデルは意味のある表現を学ばずに shortcut を使ってしまう。逆に、タスクの設計がモデルの構造と噛み合っていなければ、学習は不安定になり、得られる表現も弱くなる。

自己教師あり学習は、単にラベルなしデータを増やせばよいという話ではない。
どの情報を隠すのか。どの情報を残すのか。どの部分を予測させるのか。そして、その学習信号がモデルの構造と噛み合っているのか。これらを同時に考える必要がある。

### 1.3 本記事の問い

Masked Autoencoder (MAE) は、ViT と組み合わせることで大きな成功を収めた。画像を patch に分割し、その一部を mask し、可視 patch だけを encoder に入力する。そして、軽量な decoder によって欠損領域を復元する。この非対称な encoder-decoder 設計により、計算効率を保ちながら、強力な視覚表現を学習できる。

では、この考え方は ConvNet にもそのまま使えるのだろうか。

直感的には、画像の一部を隠して復元するだけなら、ConvNet でも同じようにできそうに見える。しかし、実際にはそう単純ではない。

ViT は画像を token の集合として扱うため、mask された token を encoder から取り除くことができる。つまり、encoder は見えている patch だけを処理すればよい。一方、ConvNet は二次元の feature map に対して畳み込みを行う。入力の一部を mask したとしても、通常の dense convolution では、**空間全体** に対して計算が行われる。

この違いは、masked image modeling において大きな問題になる。

1. mask された領域の情報を **どのように遮断するか** という問題がある。
2. mask token を使う場合、事前学習時と fine-tuning 時の **入力分布がずれる** 可能性がある。
3. ConvNet の特徴空間では、masked pretraining によって特徴が冗長化したり、**collapse** したりする可能性がある。

つまり、ViT で成功した MAE を、ConvNet にそのまま移植することはできない。

本記事で見る ConvNeXt V2 の論文は、この問題に対して、二つの方向から解決を試みている。一つは、ConvNet 向けの masked autoencoder である **FCMAE** の設計である。もう一つは、自己教師あり学習中の特徴崩壊を抑えるための **Global Response Normalization (GRN)** である。

本記事の問いは次のように整理できる。

**ConvNet で MIM を成立させるには、何を設計し直す必要があるのか。**

以降ではまず、MAE の基本思想を確認し、なぜ ConvNet への単純な移植が難しいのかを整理する。そのうえで、FCMAE と GRN がどのようにこの問題に対処しているのかを見ていく。

## 2. MAE の基本思想と ConvNet への移植問題

### 2.1 MAE は何を学習しているのか

Masked Autoencoder (MAE) は、画像の一部を隠し、残された情報から欠損領域を復元する自己教師あり学習手法である。

入力画像を patch に分割し、その一部を mask する。モデルは、見えている patch だけを手がかりに、mask された patch の内容を復元するように学習される。これは一見すると、単なる画像補完のようにも見える。

しかし、MAE の目的は、復元画像そのものをきれいに生成することではない。
重要なのは、復元タスクを通じて encoder に有用な表現を学ばせることである。

たとえば、画像の一部が隠されているとき、モデルは周囲の画素値だけを局所的に補間すればよいとは限らない。隠された領域に何が存在するかを推測するには、物体の形状、テクスチャ、背景との関係、空間的な連続性などを利用する必要がある。

つまり、MAE は「欠損領域を復元する」という pretext task を通じて、画像に含まれる構造を encoder に学習させようとする手法である。

このとき、encoder と decoder の役割は対称ではない。

encoder は、下流タスクで使いたい表現を作る部分である。一方、decoder は、事前学習時に復元タスクを解くための補助的な部分である。そのため、MAE では encoder を重くし、decoder を比較的軽くする非対称な encoder-decoder 構造が用いられる。

この設計には、二つの意味がある。

第一に、計算効率を高められる。
encoder は可視 patch のみを処理するため、mask された patch まで含めて全体を処理する必要がない。高い mask ratio を用いるほど、encoder 側の計算量を大きく削減できる。

第二に、encoder が mask された領域を直接見ることを防げる。
もし encoder が mask 領域の情報にアクセスできてしまうと、モデルは文脈から推論するのではなく、入力側に残った手がかりを使って shortcut 的に復元してしまう可能性がある。MAE では、可視 patch のみを encoder に入力することで、この問題を避けている。

このように、MAE の本質は単に「画像を隠して復元すること」ではない。
可視情報だけから欠損領域を推測させることで、encoder に意味のある表現を学ばせる点にある。

### 2.2 ViT-MAE が成立しやすかった理由

MAE が ViT と相性よく機能した理由の一つは、ViT が **画像を patch token の列として扱うこと** にある。

ViT では、入力画像を固定サイズの patch に分割し、それぞれを token として扱う。自然言語処理における単語 token と同じように、画像も token の集合として処理される。このため、mask された patch を encoder への入力から単純に取り除くことができる。

これは masked image modeling にとって非常に都合がよい。

たとえば、全 patch のうち 75% を mask する場合、encoder は残り 25% の可視 token だけを処理すればよい。mask token は encoder には入れず、decoder 側で初めて追加される。これにより、encoder は欠損領域を直接見ることなく、可視情報だけから表現を作ることになる。

この設計は、MAE の学習目的とよく噛み合っている。

encoder は、限られた可視 patch から画像全体の構造を推測する必要がある。decoder は、その encoder 表現と mask token を用いて欠損領域を復元する。下流タスクで使うのは主に encoder なので、decoder が高性能な生成器である必要はない。

むしろ、decoder が強すぎると、encoder が十分に有用な表現を学ばなくても復元ができてしまう可能性がある。したがって、MAE では軽量 decoder を用い、encoder に表現学習の負荷をかける設計になっている。

このように、ViT-MAE では次の三つが自然に実現されている。

1. mask された patch を encoder から取り除ける。
2. encoder の計算量を可視 patch 数に応じて削減できる。
3. mask token は decoder 側でのみ使われるため、encoder の入力分布が比較的単純に保たれる。

この性質は、ViT が **token-based architecture** であることに強く依存している。

したがって、MAE の成功を理解するうえでは、「画像を mask して復元する」という表面的な手続きだけでなく、ViT の token 処理と非対称 encoder-decoder 構造がどのように噛み合っていたかを見る必要がある。

### 2.3 ConvNet ではなぜ難しいのか

では、同じ masked image modeling を ConvNet に適用する場合、何が問題になるのだろうか。

最大の違いは、ConvNet が画像を **token の集合としてではなく、dense な二次元 feature map として処理する** 点にある。

ViT では、mask された patch を token 列から取り除くことができる。つまり、encoder は可視 token **だけ** を処理すればよい。一方、通常の ConvNet では、入力画像全体に対して convolution が適用される。画像の一部を mask したとしても、dense convolution は **空間全体** を走査する。

この違いにより、いくつかの問題が生じる。

第一に、**mask 領域の情報を完全に遮断しにくい。**

単純な実装としては、mask された領域を 0 で埋めたり、学習可能な mask token を入れたりする方法が考えられる。しかし、その場合でも convolution は mask 領域を含む dense な入力全体に適用される。畳み込みは近傍領域を参照するため、mask と可視領域の境界付近では、情報が混ざる可能性がある。

第二に、**計算効率の問題がある。**

MAE では、高い mask ratio を用いることで encoder の計算量を大きく削減できる。ViT であれば、可視 token だけを encoder に入力するため、mask した分だけ encoder の計算量が減る。しかし、通常の ConvNet では、mask された領域も含めて dense convolution が行われる。そのため、mask しても encoder の計算量は単純には減らない。

第三に、**事前学習時と fine-tuning 時の入力分布がずれる。**

事前学習時に入力側へ mask token を入れる場合、encoder は mask token を含む画像を処理することになる。しかし、fine-tuning や推論時には通常、mask token は存在しない。すると、事前学習時に見ていた入力分布と、下流タスクで見る入力分布が異なる。この train-time と test-time の不一致は、とくに高い mask ratio を使う場合に問題になりやすい。

第四に、**特徴空間の振る舞いが ViT と同じとは限らない。**

ConvNet は局所的な畳み込みと階層的な downsampling によって特徴を作る。これは画像認識に有効な帰納バイアスである一方、masked pretraining のもとで各チャネルがどのように分化するかは自明ではない。実際、ConvNeXt V1 に masked autoencoder を適用すると、特徴チャネルが冗長化し、feature collapse に近い挙動が観察される。

つまり、ConvNet における MIM の難しさは、単に「mask の実装が面倒」という話ではない。

MAE が ViT で成立した背景には、token を間引ける構造、可視 token のみを処理する encoder、軽量 decoder、そして mask token を decoder 側に閉じ込める設計があった。ConvNet では、これらをそのまま再現できない。

したがって、ConvNet で masked image modeling を成立させるには、MAE の表面的な手順を移植するのではなく、**ConvNet の構造に合わせて masked autoencoder 自体を設計し直す必要がある。**

そのために提案されたのが、Fully Convolutional Masked Autoencoder、すなわち **FCMAE** である。

## 3. FCMAE――ConvNet 向けに MIM を作り直す

前章で見たように、MAE の成功は単に「画像を mask して復元する」ことだけによって生じたわけではない。ViT では画像を patch token の列として扱えるため、mask された token を encoder から取り除き、可視 token だけを処理できた。この構造が、計算効率と shortcut の抑制を同時に実現していた。

一方、ConvNet は dense な二次元 feature map を前提とする。通常の convolution は空間全体に適用されるため、単純に画像の一部を mask しても、ViT-MAE のように「見えている場所だけを encoder に渡す」ことはできない。

そこで ConvNeXt V2 論文では、ConvNet 向けに masked autoencoder を作り直す。これが Fully Convolutional Masked Autoencoder、すなわち FCMAE である。

![](https://static.zenn.studio/user-upload/96a85383ed74-20260624.png)
***FCMAE framework.** Sparse convolution に基づく ConvNeXt encoder と、軽量な ConvNeXt block decoder から構成される。encoder は可視画素のみを処理し、decoder は符号化された画素と mask token を用いて画像を再構成する。損失はマスク領域に対してのみ計算される。*

FCMAE の基本構造は、masked image modeling の考え方を保ちながら、それを ConvNet の計算構造に合わせて再設計したものである。大きな特徴は、マスクされた画像を **sparse data** として扱い、encoder では **可視領域のみ** を処理する点にある。そして decoder では、encoder の出力と mask token を使って元画像を復元する。

### 3.1 マスク画像を sparse data として見る

FCMAE の重要な発想は、マスクされた画像を「値が欠けた dense image」としてではなく、**可視領域だけからなる sparse array** として見る点にある。

通常、画像に mask を適用する場合、mask された領域を 0 で埋めたり、特別な mask token に置き換えたりする。この場合、入力は依然として dense な二次元配列であり、convolution も画像全体に対して行われる。つまり、mask された場所にも計算は発生する。

しかし、masked image modeling の目的を考えると、encoder が処理すべきなのは本来、**可視領域だけ**である。mask された領域は、encoder にとって **未知** でなければならない。もし encoder が mask 領域に対応する入力を何らかの形で受け取ってしまうと、モデルは文脈から推論するのではなく、mask token や局所的な境界情報に依存する shortcut を学習する可能性がある。

FCMAE はこの問題を、マスク画像を sparse data として扱うことで回避する。

ここでいう sparse data とは、画像全体のうち、意味のある値を持つ位置だけが active site として存在するようなデータである。mask された領域は inactive site であり、encoder の**計算対象から外される。** つまり、mask 領域を事後的に 0 にするのではなく、そもそも**計算しない。**

これは、3D point cloud で使われる sparse convolution に近い発想である。点群データでは、空間全体に dense な voxel grid を作ると計算量が膨大になるため、点が存在する active site のみを処理する。FCMAE はこれと同じ見方を、masked image modeling に持ち込んでいる。

画像全体を dense な配列として見るのではなく、**可視 patch の集合** として見る。
これにより、ConvNet でも ViT-MAE に近い形で「可視部分だけを encoder に渡す」ことが可能になる。

:::details Sparse Convolution とは

#### Submanifold Sparse Convolution と ConvNeXt V2

つまり、ConvNeXt V2 における Submanifold Sparse Convolution は、単なる高速化テクニックではなく、mask token を encoder に入力する代わりに、マスク領域をそもそも計算しないための仕組みである。

この意味で、ConvNeXt V2 の FCMAE は、ViT-MAE の発想をそのまま ConvNet に移植したものではない。ConvNet の dense sliding window という性質に合わせて、masked image を sparse data として扱い直したところに設計上の工夫がある。

入力画像を

X\in\mathbb{R}^{H\times W}
$$

とし、画素位置 $(i,j)$ の値を $X_{i,j}$ と書く。

また、非ゼロ、あるいは意味のある画素の集合を active site と呼び、

$$
\mathcal{A}\subset \{1,\dots,H\}\times\{1,\dots,W\}
$$

とする。

$3\times3$ の畳み込みカーネルを

$$
K=
\begin{pmatrix}
w_{-1,-1} & w_{-1,0} & w_{-1,1} \\
w_{0,-1} & w_{0,0} & w_{0,1} \\
w_{1,-1} & w_{1,0} & w_{1,1}
\end{pmatrix}
$$

とする。

通常の畳み込みでは、各画素 $(i,j)$ に対して

$$
\begin{aligned}
Y_{i,j}={}&w_{-1,-1}X_{i-1,j-1} &+{}& w_{-1,0}X_{i-1,j} &+{}& w_{-1,1}X_{i-1,j+1}&+{}&\\
&w_{0,-1}X_{i,j-1}     &+{}& w_{0,0}X_{i,j}     &+{}& w_{0,1}X_{i,j+1}&+{}&\\
&w_{1,-1}X_{i+1,j-1}  &+{}& w_{1,0}X_{i+1,j}  &+{}& w_{1,1}X_{i+1,j+1}
\end{aligned}
$$

を計算する。

一方、**Submanifold Sparse Convolution** では、出力を計算する位置を active site に限定する。すなわち、$(i,j)\in\mathcal{A}$ のときだけ $Y_{i,j}$ を計算し、$(i,j)\notin\mathcal{A}$ の画素は計算しない。

さらに、近傍画素についても active site の値だけを使う。したがって、

$$
Y_{i,j}=\sum_{\substack{a,b\in\{-1,0,1\}\\(i+a,j+b)\in\mathcal{A}}}w_{a,b}X_{i+a,j+b},\qquad(i,j)\in\mathcal{A}
$$

となる。

$(i,j)\notin\mathcal{A}$ の場合は、$Y_{i,j}=0$ とみなすか、実装上はそもそも $Y_{i,j}$ を保存しない。

つまり、Submanifold Sparse Convolution は次のように書ける。

$$
Y_{i,j}=
\begin{cases}
\displaystyle
\sum_{\substack{a,b\in\{-1,0,1\}\\(i+a,j+b)\in\mathcal{A}}}w_{a,b}X_{i+a,j+b}&(i,j)\in\mathcal{A},\\
\qquad 0&(i,j)\notin\mathcal{A}.
\end{cases}
$$

通常の sparse convolution との違いは、出力位置の扱いにある。

通常の sparse convolution では、近傍に active site が1つでもあれば、その位置を新たな active site として出力する。そのため、層を重ねると active site が周囲に広がっていく。

一方、Submanifold Sparse Convolution では、

$$
\mathcal{A}_{\mathrm{out}}=\mathcal{A}_{\mathrm{in}}
$$

である。

つまり、入力で active だった画素にだけ出力を作る。近傍の active 画素から情報は集約するが、inactive な位置に新しい active site は作らない。

したがって、Submanifold Sparse Convolution は **active site 上だけで行う畳み込み** と考えればよい。カーネル重み $w_{a,b}$ 自体は通常の畳み込みと同じように存在するが、実際に計算されるのは、出力位置と入力近傍の両方が active な場合だけである。

:::

### 3.2 Sparse convolution encoder

FCMAE では、encoder として ConvNeXt を用いる。ただし、通常の dense convolution のままではなく、事前学習時には convolution layer を sparse convolution として扱う。

これにより、encoder は active site、つまり可視領域に対してのみ計算を行う。mask された領域は inactive site であり、encoder の計算には参加しない。

この設計には三つの意味がある。

1. **mask 領域からの情報漏洩を防ぎやすくなる。**
   encoder は可視領域だけを処理するため、mask された領域の値や mask token に依存する shortcut を使いにくい。これは、MAE の本来の目的である「見えている情報から見えていない部分を推測する」という学習信号に合っている。

2. **事前学習時と fine-tuning 時の入力分布のずれを小さくできる。**
   入力側に mask token を入れる方法では、事前学習時の encoder は mask token を含む画像を処理する。一方、fine-tuning や推論時には mask token は存在しない。この train-time / test-time mismatch は、高い mask ratio を用いるほど大きくなる。FCMAE では、mask された位置を encoder に入れないため、この問題を緩和できる。

3. **masking による計算削減が可能になる。**
   通常の dense convolution では、mask された領域にも計算が発生する。しかし sparse convolution を使えば、可視領域だけを処理できる。これは、ViT-MAE が可視 token のみを encoder に入れることで計算効率を得ていたことに対応している。

論文では、sparse convolution の有無が fine-tuning accuracy に大きく影響することが示されている。

|               設計 | ImageNet-1K fine-tuning accuracy |
| ---------------: | -------------------------------: |
| w/o Sparse conv. |                             79.3 |
|  w/ Sparse conv. |                             83.7 |

ここから、単に画像を mask して復元させるだけでは不十分であり、encoder がどの領域を処理するかを明示的に設計する必要があると分かる。

なお、論文では sparse convolution layer は fine-tuning 時に標準的な dense convolution に **戻せる** と説明されている。つまり、**sparse convolution は主に事前学習時の仕組み** であり、下流タスクでは通常の ConvNet として扱える。

これは実用上も重要である。事前学習のために特殊な入力形式を使っても、fine-tuning や推論時に複雑な処理が必要になると使いにくい。FCMAE では、事前学習時には sparse convolution によって MIM を成立させ、fine-tuning 時には通常の ConvNeXt として扱えるようにしている。

### 3.3 軽量 decoder と masked reconstruction

FCMAE の設計は、単に ConvNeXt に decoder を付けて画像を復元させる、というものではない。重要なのは、どこを mask し、encoder に何を見せ、decoder にどこまで任せるかである。

まず、FCMAE では入力画像の一部を patch 単位で mask する。具体的には、元画像を $32 \times 32$ patch に分割し、そのうち 60% をランダムに除去する。ConvNeXt は階層的な構造を持ち、stage が進むにつれて特徴マップの解像度が下がっていく。そのため **FCMAE では、mask を最後の stage の解像度で生成し、それをより細かい解像度へ再帰的に upsample することで、各 stage に対応する mask を構成している。**

これにより、encoder は人工的な mask token を直接見る必要がない。mask token を encoder に入力する設計では、事前学習時には存在するが fine-tuning 時には存在しない人工的な入力が混入するため、入力分布のずれが生じうる。FCMAE では mask token を decoder 側で用いることで、この種の artifact を encoder から切り離している。

encoder には ConvNeXt を用い、decoder には軽量な ConvNeXt block を用いる。ここでも重要なのは、decoder を過度に強くしないことである。

MAE 系の自己教師あり学習では、下流タスクで使いたいのは基本的に **encoder** である。decoder は、事前学習時に reconstruction loss を定義するための **補助的なモジュール**であり、最終的な表現を直接提供するわけではない。

そのため、decoder が強すぎると問題が起こる可能性がある。もし decoder だけで復元タスクをうまく解けてしまうなら、encoder は下流タスクに有用な表現を十分に学習しなくてもよい。つまり、reconstruction loss が小さくなっても、表現学習としては強いとは限らない。

FCMAE では、この点を踏まえて、**単一の ConvNeXt block** に基づく軽量 decoder を採用している。論文では、より複雑な hierarchical decoder や transformer decoder も検討されているが、最終的には軽量な ConvNeXt block decoder が、fine-tuning accuracy と事前学習時間のバランスに優れていた。

再構成対象は、元画像を patch-wise に正規化した画像である。損失には mean squared error (MSE) を用い、mask された patch に対してのみ計算する。

ここで重要なのは、loss を画像全体に対して計算するのではなく、**mask 領域に限定** する点である。可視領域はすでに encoder に与えられているため、そこを復元させても学習信号としては弱い。モデルに求めたいのは、見えていない領域を周囲の文脈や画像構造から推測することである。したがって、reconstruction loss は mask された領域に対してのみ計算される。

FCMAE の設計を整理すると、次のようになる。

* 入力画像を $32 \times 32$ patch 単位で分割し、その 60% を mask する。
* 階層的な ConvNet に対応するため、mask を各 stage の解像度に合わせて展開する。
* mask された画像を sparse data として扱う。
* encoder は sparse convolution によって、主に可視領域を処理する。
* decoder は encoder の出力と mask token を用いて欠損領域を復元する。
* reconstruction loss は mask 領域に対してのみ計算する。

この構成により、ConvNet でも masked image modeling を実行できるようになる。

FCMAE は、ConvNet に MIM を適用するための枠組みを与えた。しかし、ConvNeXt V1 に FCMAE を組み合わせるだけでは、ViT-MAE のような強い表現学習にはまだ **届かない。** 実際、FCMAE によって教師あり 100 epoch baseline は上回るものの、ConvNeXt V1 の強い教師あり baseline と比べると、改善は限定的である。

|         設定 | ImageNet-1K fine-tuning accuracy |
| ---------: | -------------------------------: |
| Sup. 100ep |                             82.7 |
| Sup. 300ep |                             83.8 |
|      FCMAE |                             83.7 |

この結果は、FCMAE が有効である一方で、それだけでは十分でないことを示している。

問題は、masked autoencoder の構造だけではなかった。**ConvNet 側の特徴空間そのものにも、masked pretraining において不都合な挙動が生じていた。**

次章では、この問題である **feature collapse** を見る。

## 4. FCMAE だけでは足りない――feature collapse の発見

前章では、ConvNet 向けの masked autoencoder として FCMAE を見た。

FCMAE は、マスクされた画像を sparse data として扱い、encoder では可視領域のみを処理する。これにより、ViT-MAE が持っていた「可視 token だけを encoder に入力する」という性質を、ConvNet に近い形で再現している。

しかし、FCMAE を導入すればそれだけで ConvNet における masked image modeling が完全に解決するわけではない。

論文では、ConvNeXt V1 に FCMAE を組み合わせることで、教師あり 100 epoch baseline よりは良い結果が得られる。一方で、ConvNeXt V1 の強い教師あり baseline を明確に上回るところまでは到達していない。

つまり、問題は masked autoencoder の構造だけではなかった。
ConvNet 側の特徴空間にも、masked pretraining と噛み合わない挙動が存在していた。

その代表的な現象が、**feature collapse** である。

![](https://static.zenn.studio/user-upload/67e2aeb27feb-20260624.png)
***Feature activation visualization.** 各 feature channel の activation map を小さな正方形として可視化する。見やすさのため、各可視化では 64 channel を表示している。ConvNeXt V1 モデルは feature collapse の問題を抱えており、これはチャネル間に冗長な activation（dead または saturated neuron）が存在することによって特徴づけられる。*

### 4.1 Activation map から見える collapse

論文では、FCMAE で事前学習した ConvNeXt-Base の activation map を可視化し、特徴チャネルに冗長な挙動が生じていることを報告している。

具体的には、多くの feature map がほとんど反応していない、あるいは逆に飽和したような反応を示している。言い換えれば、チャネルごとに異なる情報を捉えているというより、似たような反応や使われていない反応が増えている。

これは、自己教師あり学習においてはかなり重要な問題である。

教師あり分類であれば、最終的にクラス識別に必要な特徴へ表現が寄っていくこと自体は自然である。モデルは、与えられたラベルを予測するために有用な特徴を強調し、それ以外の情報を捨てる。その意味で、分類に特化した特徴が形成されることは、必ずしも悪いことではない。

しかし、自己教師あり事前学習では事情が異なる。

SSL の目的は、特定のラベル集合だけに最適化された特徴ではなく、下流タスクへ転移しやすい汎用的な表現を学習することである。そのためには、**特徴チャネルが十分に多様であり、画像中のさまざまな構造を分担して捉えていることが望ましい。**

もし多くのチャネルが dead feature になっていたり、似たような activation ばかりを出していたりするなら、モデル容量を十分に使えていない可能性がある。表現空間が狭くなり、下流タスクに有用な情報を取りこぼす危険がある。

ここで観察されている feature collapse は、完全にすべての表現が一点に潰れるという意味ではない。contrastive learning で問題になるような、**すべての入力が同じ表現になる collapse とは少し違う。**

むしろここでの問題は、チャネル方向の多様性が失われることである。
各チャネルが異なる特徴を担うのではなく、一部のチャネルが死んだり、複数のチャネルが似たような反応を示したりする。その結果、特徴表現が冗長になり、masked pretraining によって期待される表現学習が十分に進まなくなる。

論文では、この現象が主に ConvNeXt block 内の dimension-expansion MLP 層で見られたと報告している。これは、後に GRN をどこへ挿入するかという設計にも関係する。

Feature activation の可視化は ConvNeXt V1 に FCMAE を載せるだけでは、特徴チャネルが十分に分化せず、SSL に適した表現空間が形成されにくいことを示唆する重要な結果である。

### 4.2 Feature cosine distance による定量解析

ただし、activation map の可視化だけでは、feature collapse を定性的に眺めているにすぎない。そこで論文では、チャネル間の特徴多様性を定量化するために、feature cosine distance analysis を行っている。

activation tensor を $X\in\mathbb{R}^{H\times W\times C}$ とする。ここで、$H$ と $W$ は空間方向のサイズ、$C$ はチャネル数である。また、$X_i\in\mathbb{R}^{H\times W}$ を $i$ 番目のチャネルの feature map とする。

各 feature map $X_i$ を $HW$ 次元のベクトルとみなし、チャネル間の平均 pair-wise cosine distance を次のように計算する。

$$
\frac{1}{C^2}\sum_i^C\sum_j^C \frac{1-\cos(X_i,X_j)}{2}
$$

この値は、チャネル同士がどれだけ異なる反応をしているかを表す指標として解釈できる。

cosine similarity が高い場合、二つのチャネルは似た方向の activation pattern を持つ。つまり、似たような特徴を表している可能性が高い。逆に cosine distance が大きい場合、チャネル同士は異なる反応をしており、特徴の多様性が高いと見なせる。

したがって、この平均 cosine distance が小さいほど、feature redundancy が大きい。
多くのチャネルが似たような反応をしており、表現が冗長になっていることを意味する。

![](https://static.zenn.studio/user-upload/b1761794746b-20260624.png)
***Feature cosine distance analysis.** アーキテクチャによって総層数が異なるため、normalized layer index に対して distance value を plot している。*

この図では、いくつかのモデルについて、層ごとの feature cosine distance が比較されている。ここで重要なのは、**FCMAE で事前学習された ConvNeXt V1 が、層が深くなるにつれて feature diversity を大きく失っている** 点である。

教師あり学習された ConvNeXt でも、最終層付近では feature diversity が低下する。これはある程度自然である。分類モデルでは、最終的にクラス識別に有用な特徴へ表現を集約していくためである。

一方、FCMAE で事前学習された ConvNeXt V1 では、より広い範囲で feature diversity の低下が見られる。これは、単なる分類タスクへの適応ではなく、masked pretraining の過程で特徴チャネルが **冗長化** していることを示唆する。

対照的に、MAE で事前学習された ViT では、特徴の多様性が比較的保たれている。これは、ViT-MAE が masked image modeling と相性よく機能していることと整合的である。

この解析により、問題の焦点がかなり明確になる。

FCMAE は、ConvNet に masked image modeling を適用するための入力・計算構造を与えた。
しかし、ConvNeXt V1 のままでは、masked pretraining 中に特徴チャネルの多様性が失われやすい。

つまり、ConvNet で MIM を成立させるには、masked autoencoder の構造だけでなく、**アーキテクチャ内部の特徴応答も設計する必要がある。**

### 4.3 SSL において feature diversity が重要な理由

では、なぜ feature diversity がそれほど重要なのだろうか。

自己教師あり学習では、モデルは明示的なクラスラベルを与えられない。その代わりに、pretext task を通じて画像の構造を学ぶ。masked image modeling であれば、隠された領域を文脈から復元することが学習信号になる。

このとき、モデルが学習すべき情報は一種類ではない。

画像には、局所的なテクスチャ、エッジ、形状、物体の配置、背景との関係など、さまざまな構造が含まれている。下流タスクも、分類、検出、セグメンテーションのように多様である。そのため、事前学習で得られる表現には、特定の一方向に偏りすぎない多様性が求められる。

特徴チャネルが十分に多様であれば、各チャネルは異なる視覚的手がかりを捉えやすくなる。あるチャネルは局所的なテクスチャに反応し、別のチャネルは形状や領域構造に反応するかもしれない。このように、**特徴空間の中に多様な反応が存在することで、下流タスクに応じて必要な情報を取り出しやすくなる。**

逆に、feature collapse が起きると、モデルは容量を十分に使えない。
多くのチャネルが似た反応をしているなら、チャネル数が多くても実質的な表現の次元は小さくなる。dead feature が増えるなら、そもそも使われていない容量が増える。

これは、SSL において特に問題になる。

教師あり学習では、ラベルが明確な学習方向を与える。モデルは、分類に有効な特徴を強めるように最適化される。その結果として一部の特徴が抑制されても、分類性能が高ければ目的は達成される。

しかし、SSL では、特定のラベル空間に閉じた特徴ではなく、下流タスクに**広く使える表現を作りたい。** そのため、事前学習の段階で特徴が過度に冗長化すると、転移性能に悪影響を与える可能性がある。

この観点から見ると、ConvNeXt V1 + FCMAE の問題は、単に fine-tuning accuracy が少し足りないという話ではない。

より本質的には、masked pretraining に対して ConvNeXt V1 の特徴空間が十分に適応していなかった、ということである。FCMAE によって入力と計算の問題は解決されつつあったが、**内部表現の多様性を保つ仕組みが不足していた。**

そこで導入されるのが、**Global Response Normalization (GRN)** である。

GRN の目的は、チャネル間の feature competition を強め、feature diversity を維持することである。次章では、GRN がどのように ConvNeXt block に組み込まれ、FCMAE と組み合わせることで masked image modeling を有効にしているのかを見る。

---

## 5. GRN――SSL に適した ConvNeXt への修正

前章では、ConvNeXt V1 に FCMAE を適用したとき、特徴チャネルの多様性が失われる feature collapse が生じることを見た。

これは、FCMAE の設計が無意味だったということではない。
むしろ、FCMAE によって ConvNet でも masked image modeling を行うための入力・計算構造はかなり整理された。マスク画像を sparse data として扱い、encoder では可視領域のみを処理する。この設計により、ViT-MAE に近い形で、見えている情報だけから欠損領域を推測する学習が可能になる。

しかし、それだけでは不十分だった。

問題は、ConvNeXt V1 の内部表現が masked pretraining に対して十分に適応していなかったことにある。とくに、ConvNeXt block 内の dimension-expansion MLP 層で、特徴チャネルが冗長化し、多様性を失う挙動が観察された。

そこで導入されるのが、Global Response Normalization、すなわち GRN である。

GRN の目的は、チャネル間の feature competition を強め、feature diversity を維持することである。言い換えれば、各チャネルが似たような反応に潰れるのを防ぎ、**それぞれのチャネルが異なる特徴を担いやすくするための仕組み** である。

![](https://static.zenn.studio/user-upload/4109b4bd1a02-20260624.png)
***ConvNeXt Block Designs.** ConvNeXt V2 では、dimension-expansion MLP layer の後に GRN layer を追加し、冗長になる LayerScale を削除する。*

### 5.1 GRN はどこに入るのか

GRN は、ConvNeXt block の中でも、dimension-expansion MLP layer の後に挿入される。

この位置は重要である。
前章で見たように、feature collapse は主に ConvNeXt block 内の MLP 層で観察されていた。ConvNeXt block では、まず depthwise convolution によって空間的な情報を処理し、その後、MLP 的なチャネル変換によって特徴を混合・拡張する。この dimension-expansion 部分は表現力を高める一方で、masked pretraining のもとでは一部のチャネルが冗長化しやすい。

GRN は、この高次元化された特徴に対して適用される。
直感的には、MLP によって拡張された各チャネルの反応を見比べ、相対的にどのチャネルが強く反応しているかを評価する。そして、その情報を使って元の feature map を調整する。これにより、チャネル間に競合関係が生まれ、似たようなチャネルばかりが増えることを抑える。

ConvNeXt V2 では、この GRN を ConvNeXt block に組み込むことで、自己教師あり学習により適した block design へと変更している。また、GRN の導入により、ConvNeXt V1 で用いられていた LayerScale は不要になり、削除されている。

ここで重要なのは、GRN が大規模な構造変更ではないという点である。

ConvNeXt V2 は、ConvNeXt V1 を根本から作り直したモデルではない。変更としては比較的小さい。しかし、その変更は masked pretraining 中に観察された feature collapse という具体的な問題に対応している。

つまり、GRN は単なる性能改善のための部品ではなく、FCMAE による自己教師あり学習を ConvNeXt 上で成立させるためのアーキテクチャ側の修正である。

### 5.2 GRN は何をしているのか

GRN の処理は、大きく三つの段階に分けられる。

1. 各チャネルの global response を集約する。
2. チャネル間で response を正規化する。
3. その正規化された値を使って元の feature map を補正する。

入力特徴を $X\in\mathbb{R}^{H\times W\times C}$ とする。ここで、$H$ と $W$ は空間サイズ、$C$ はチャネル数である。また、$X_i\in\mathbb{R}^{H\times W}$ を $i$ 番目のチャネルの feature map とする。

まず、GRN は各チャネル $X_i$ の大きさを global に集約する。論文では、global average pooling ではなく、L2 norm による集約が有効であることが示されている。

$$
\mathcal{G}(X)={\|X_1\|,\|X_2\|,\ldots,\|X_C\|}\in\mathbb{R}^C
$$

ここで、$\|X_i\|$ は $i$ 番目のチャネルの feature map 全体の L2 norm である。
これは、そのチャネルが空間全体でどれくらい強く反応しているかを表す scalar と見なせる。

次に、各チャネルの response を、全チャネルの response と比較する。

$$
\mathcal{N}(\|X_i\|)=\frac{\|X_i\|}{\sum_{j=1}^{C}\|X_j\|}
$$

これは、$i$ 番目のチャネルが、全チャネルの中で相対的にどの程度強く反応しているかを表している。絶対的な大きさではなく、チャネル間の相対的な response を見る点が重要である。

この操作によって、チャネル間に competition が生じる。
あるチャネルの response が強いかどうかは、そのチャネル単独ではなく、他のチャネルとの比較によって決まる。これにより、すべてのチャネルが似たような反応をするのではなく、チャネルごとの役割分担が促される。

最後に、この正規化された response を使って、元の feature map を補正する。

$$
\tilde{X}_i = X_i \odot \mathcal{N}(\mathcal{G}(X)_i)
$$

つまり、各チャネルの feature map に、そのチャネルの相対的な global response を掛ける。これにより、チャネルごとの反応が調整される。

ただし、実際の GRN layer では、最適化を安定させるために learnable parameter である $\gamma$ と $\beta$ を加え、さらに residual connection を持たせている。

最終的には、GRN は初期状態ではほぼ identity function として振る舞い、学習の中で徐々に feature response を調整する。これは重要である。最初から強い正規化をかけるのではなく、学習に応じて必要な補正を獲得できるからである。

GRN は、Batch Normalization や Layer Normalization のように単に分布を整えるための層というより、チャネル間の相対的な反応を使って feature competition を導入する層と見る方が分かりやすい。

この違いは、masked pretraining では特に重要である。

BN は batch axis に依存するため、マスクされた入力や小さい batch size の設定では扱いにくい。LN は各サンプル内で特徴を標準化するが、チャネル間の global response を明示的に競合させるわけではない。SE や CBAM のような gating module もチャネルや空間方向の重み付けを行うが、追加の parameter layer を必要とする。

一方、GRN は非常に単純で、計算量やパラメータをほとんど増やさずに、チャネル間の competition を導入できる。

論文の ablation でも、GRN が masked pretraining において有効であることが示されている。

![](https://static.zenn.studio/user-upload/4402cf60ff5e-20260624.png)
***GRN ablations with ConvNeXt-Base.** ImageNet-1K における fine-tuning accuracy 結果。最終的に採用した設定は `gray` で示す。*

この表では、global feature aggregation、normalization function、residual connection の有無、他の normalization layer や gating method との比較が検討されている。

ここで見るべき点は、GRN が単に「何らかの正規化を入れればよい」という話ではないことである。L2 norm によって各チャネルの global response を測り、それをチャネル間で相対化し、residual connection 付きで元の特徴に戻す。この組み合わせが、masked pretraining における feature diversity の維持に効いている。

特に重要なのは、GRN が pre-training と fine-tuning の両方で必要とされる点である。事前学習時だけ GRN を使い、fine-tuning 時に取り除くと性能は下がる。また、事前学習時には使わず、fine-tuning 時だけ新たに GRN を追加しても十分には機能しない。

これは、GRN が単なる fine-tuning 用の補正ではないことを意味する。
GRN は、事前学習の過程で形成される特徴空間そのものに関わっている。つまり、FCMAE によって学習される表現が、GRN を含む ConvNeXt V2 block の中で形成されることに意味がある。

### 5.3 FCMAE と GRN は組み合わせて効く

ここまで見ると、ConvNeXt V2 の本質はかなり明確になる。

FCMAE は、ConvNet で masked image modeling を行うための学習フレームワークである。
GRN は、その masked pretraining 中に特徴チャネルが潰れることを防ぐためのアーキテクチャ修正である。

重要なのは、この二つが単独で効いているのではなく、組み合わせて効いている点である。

![](https://static.zenn.studio/user-upload/5f38fd5956c7-20260624.png)
***Co-design matters.** アーキテクチャと学習フレームワークを共同設計し、それらを組み合わせて用いると、masked image pre-training は ConvNeXt に対して有効になる。*

この表では、FCMAE と GRN の効果が比較されている。

ConvNeXt V1 に FCMAE を適用すると、一定の改善は得られる。しかし、それだけでは改善は限定的である。一方、GRN を導入した ConvNeXt V2 も、教師あり学習の設定では効果が比較的小さい。つまり、FCMAE だけでも、GRN だけでも、決定的な改善にはならない。

大きな改善が現れるのは、FCMAE と GRN を **組み合わせたとき** である。

これは、この論文の中心的な主張である。

masked image modeling を ConvNet に適用するには、学習フレームワークだけを変えても不十分である。
同時に、アーキテクチャ側も masked pretraining に適した形へ修正する必要がある。

逆に言えば、アーキテクチャだけを少し改良しても、それが自己教師あり学習の学習信号と噛み合っていなければ、大きな効果は出ない。

この関係は、自己教師あり学習を考えるうえで非常に重要である。

SSL 手法は、しばしば「どの pretext task を使うか」という観点で語られる。contrastive learning なのか、masked image modeling なのか、distillation 系なのか、という分類である。もちろん、それは重要である。

しかし、実際には pretext task だけで性能が決まるわけではない。

同じ masked image modeling であっても、ViT と ConvNet では適した実装が異なる。
ViT では token を間引く設計が自然に成立する。一方、ConvNet では sparse convolution によって可視領域のみを処理する工夫が必要になる。さらに、ConvNet の内部では feature collapse が起こりやすく、それを抑えるために GRN のような仕組みが必要になる。

つまり、SSL の性能は、学習信号とアーキテクチャの **相互作用** によって決まる。

この観点から見ると、ConvNeXt V2 は単なる ConvNeXt の改良版ではない。
ConvNeXt V2 は、FCMAE という自己教師あり学習フレームワークと組み合わせることを前提に、ConvNeXt を再設計したモデルである。

言い換えれば、ConvNeXt V2 + FCMAE の価値は、次の一点に集約できる。

**自己教師あり学習を成功させるには、pretext task とアーキテクチャを同時に設計する必要がある。**

これは、ViT-MAE の成功を ConvNet に移植しようとしたときに初めて明確になった教訓でもある。MAE は強力な SSL 手法だが、その強さは ViT の token-based architecture と深く結びついていた。ConvNet で同じ学習信号を使うには、ConvNet 側の構造を見直す必要があった。

FCMAE は、ConvNet で MIM を行うための入力と計算の設計である。
GRN は、masked pretraining 中の特徴空間を安定させるための設計である。
この二つが組み合わさることで、ConvNet でも masked image modeling が有効に機能する。

したがって、ConvNeXt V2 の論文を読むうえで重要なのは、最終的な精度だけではない。

むしろ重要なのは、なぜ単純な ConvNeXt + MAE では不十分だったのか、何を観察し、どの問題に対してどの設計を入れたのか、という流れである。

この流れは、自己教師あり学習を実際に別のデータ・別のモデルへ適用するときにも参考になる。
SSL は、既存手法をそのまま載せればよいものではない。対象データ、モデル構造、mask の扱い、特徴空間の挙動を見ながら、学習信号とアーキテクチャを一体として設計する必要がある。

## 6. この論文から何を学ぶべきか

ここまで、ConvNeXt V2 と FCMAE の設計を、自己教師あり学習の観点から見てきた。

この論文の価値は、単に ConvNet の性能を上げたことだけにあるわけではない。もちろん、ImageNet 分類、COCO 物体検出、ADE20K セグメンテーションといった下流タスクで高い性能を示したことは重要である。しかし、それ以上に重要なのは、masked image modeling を ConvNet に適用する際に、何が問題になり、どのような設計変更が必要だったのかを明確に示した点にある。

ViT-MAE の成功を見ると、画像の一部を mask し、それを復元させるだけで強い表現が学習できるように見える。しかし、ConvNeXt V2 の論文が示しているのは、それほど単純な話ではないということである。

自己教師あり学習は、pretext task だけで完結するものではない。
どの情報を隠すのか。どの情報を encoder に渡すのか。decoder にどれくらいの役割を持たせるのか。特徴空間が学習中にどのように振る舞うのか。そして、それらがモデルの構造と噛み合っているのか。

これらを同時に考えなければ、SSL は期待通りに機能しない。

### 6.1 SSL 手法はアーキテクチャ非依存ではない

ConvNeXt V2 の論文から得られる最も重要な教訓は、自己教師あり学習手法はアーキテクチャ非依存ではない、という点である。

MAE は、ViT と組み合わせることで大きな成功を収めた。ViT では、画像を patch token の列として扱うため、mask された token を encoder から取り除くことができる。encoder は可視 token だけを処理し、mask token は decoder 側で初めて使われる。この設計により、計算効率を高めながら、encoder が欠損領域を直接見ることを防げる。

しかし、この設計は ViT の token-based architecture に強く依存している。

ConvNet では、画像は dense な二次元 feature map として処理される。通常の convolution は空間全体に適用されるため、mask された領域を単純に取り除くことはできない。mask token を入力に入れる方法も考えられるが、その場合、事前学習時と fine-tuning 時の入力分布がずれる。また、mask された領域にも計算が発生し、ViT-MAE のような計算効率も得にくい。

そのため、ConvNeXt V2 論文では、まず FCMAE によって ConvNet 向けの masked autoencoder を設計している。マスク画像を sparse data として扱い、encoder では可視領域だけを処理する。これにより、ConvNet でも「見えている情報だけを使って、見えていない領域を推測する」という MAE の本質に近づけている。

しかし、それでも十分ではなかった。

ConvNeXt V1 に FCMAE を組み合わせると、feature collapse が観察された。つまり、特徴チャネルの多様性が失われ、dead feature や冗長な activation が増えてしまう。これは、masked pretraining において ConvNet の内部表現が必ずしも自然に分化するわけではないことを示している。

そこで GRN が導入される。GRN は、チャネル間の global response を比較し、feature competition を強めることで、feature diversity を維持する。これにより、FCMAE による masked pretraining が ConvNeXt 上でより有効に機能するようになる。

この流れは、SSL の設計を考えるうえで重要である。

同じ masked image modeling であっても、ViT と ConvNet では必要な設計が異なる。
同じ reconstruction loss を使っていても、encoder が何を見ているか、mask 領域の情報が漏れていないか、特徴空間が collapse していないかは、アーキテクチャによって変わる。

したがって、SSL 手法を別のモデルへ移植するときには、「同じ objective を使えば同じように効く」と考えるべきではない。pretext task とアーキテクチャの相性を確認し、必要なら学習フレームワークとモデル構造を同時に設計し直す必要がある。

### 6.2 スケーリングと転移性能

ConvNeXt V2 + FCMAE の有効性は、単一のモデルサイズに限られていない。論文では、Atto から Huge まで、さまざまなサイズの ConvNeXt V2 に対して FCMAE 事前学習を適用し、教師あり baseline と比較している。

![](https://static.zenn.studio/user-upload/ee138c2af42a-20260624.png)
***ConvNeXt V2 model scaling.** 我々の fully convolutional masked autoencoder framework を用いて事前学習された ConvNeXt V2 モデルは、幅広いモデルサイズにわたって、従来バージョンよりも大幅に優れた性能を示す。*

この図が示しているように、FCMAE で事前学習された ConvNeXt V2 は、広いモデルサイズにわたって ConvNeXt V1 を上回っている。これは、提案手法が特定のサイズだけに偶然効いたのではなく、モデルスケールを変えても一貫して有効であることを示している。

この点は重要である。

小さいモデルでだけ効く手法は、軽量モデル向けの工夫としては有用かもしれない。しかし、大きなモデルにしたときに効果が消えるなら、表現学習の基盤としてはやや弱い。逆に、大きなモデルでだけ効く手法は、計算資源が限られた環境では使いにくい。

ConvNeXt V2 + FCMAE は、比較的軽量なモデルから大規模モデルまで、一貫して改善を示している。このことは、FCMAE と GRN の組み合わせが、単なる局所的なチューニングではなく、ConvNet における masked pretraining の基本的な問題に対応していることを示唆している。

また、改善は ImageNet 分類だけに閉じていない。

論文では、COCO における物体検出・インスタンスセグメンテーション、ADE20K における semantic segmentation でも評価が行われている。これらの結果でも、ConvNeXt V2 + FCMAE は教師あり学習された ConvNeXt V1 を上回り、Swin Transformer 系のモデルとも競合する性能を示している。

ここで重要なのは、個々の数値そのものよりも、改善が下流タスクへ転移している点である。

自己教師あり事前学習の目的は、pretraining task の loss を下げることではない。下流タスクに有用な表現を得ることである。したがって、reconstruction loss が下がるかどうかだけでは、SSL の成功は判断できない。

FCMAE と GRN の共同設計によって得られた表現が、分類だけでなく、検出やセグメンテーションにも有効であるなら、それはより汎用的な視覚表現が学習されていることを示す材料になる。

ただし、この論文を読むときに、SOTA の数値だけを追うと本質を見失いやすい。

ImageNet-22K intermediate fine-tuning による 88.9% top-1 accuracy や、既存 MIM 手法との比較はもちろん重要である。しかし、本記事の文脈でより重要なのは、最終精度の高さそのものではなく、そこに至るまでの設計の流れである。

単純な ConvNeXt + MAE では不十分だった。
FCMAE によって、ConvNet 向けに masked autoencoder を作り直した。
それでも feature collapse が残った。
GRN によって、masked pretraining 中の feature diversity を維持した。
その結果、ConvNet でも MIM が有効に機能するようになった。

この流れこそが、ConvNeXt V2 論文の中心である。

### 6.3 実用上の示唆

最後に、この論文から実用上どのような示唆が得られるかを整理する。

第一に、ConvNet で masked image modeling を行う場合、mask token を入力に入れるだけでは不十分である。

ViT-MAE では、mask token を encoder に入れず、decoder 側で扱うことができた。しかし ConvNet では、dense convolution の性質上、mask された領域の扱いが難しい。入力を 0 で埋めたり、mask token を入れたりするだけでは、情報漏洩、計算効率、train-time / test-time mismatch の問題が残る可能性がある。

FCMAE のように、mask された画像を sparse data として扱い、encoder が可視領域のみを処理する設計は、この問題に対する一つの有効な解である。

第二に、reconstruction loss だけを見ても、表現学習の良し悪しは判断できない。

masked image modeling では、復元誤差が下がると学習がうまくいっているように見える。しかし、下流タスクで重要なのは、きれいに復元できることではなく、encoder が有用な表現を獲得していることである。

decoder が強すぎれば、encoder が十分な表現を学ばなくても復元できてしまうかもしれない。逆に、復元 loss がある程度下がっていても、内部では feature collapse が起きている可能性がある。

したがって、SSL の評価では、pretraining loss だけでなく、fine-tuning 性能、転移性能、特徴空間の解析を合わせて見る必要がある。

第三に、feature diversity は自己教師あり学習において重要な観察対象である。

教師あり分類では、最終的にクラス識別に有用な特徴へ表現が集約されることは自然である。しかし SSL では、下流タスクに広く使える表現を学習したい。そのため、事前学習中に特徴チャネルが冗長化していないか、dead feature が増えていないかを確認することは重要である。

ConvNeXt V2 論文では、activation visualization と feature cosine distance analysis によって、この問題を具体的に捉えている。このような特徴空間の解析は、SSL 手法を別のデータセットや別のモデルへ適用するときにも参考になる。

第四に、SSL では「手法を選ぶ」だけでなく、「手法とモデルの噛み合わせを見る」必要がある。

MAE、DINO、SimMIM、iBOT など、自己教師あり学習には多くの手法がある。しかし、どの手法がよいかは、データ、モデル、計算資源、下流タスクによって変わる。特に masked image modeling のように、入力の一部を隠す手法では、モデルがその mask をどのように処理するかが重要になる。

ConvNeXt V2 + FCMAE の例は、この点をよく示している。

ViT で成功した MAE を ConvNet にそのまま載せるのではなく、ConvNet に合わせて sparse convolution encoder を設計し、さらに GRN によって特徴空間の崩れを抑えた。これは、SSL を実際に使ううえで非常に現実的な教訓である。

自己教師あり学習は、ラベルなしデータを使えるという点で強力である。
しかし、それは「ラベルなしデータを入れれば勝手に良い表現が得られる」という意味ではない。

どの情報を隠すのか。
どの情報を encoder に渡すのか。
decoder にどれだけの能力を持たせるのか。
特徴空間が collapse していないか。
その表現は下流タスクへ転移するのか。

これらを確認しながら設計する必要がある。

ConvNeXt V2 + FCMAE の価値は、単に ConvNet の精度を上げたことではない。
自己教師あり学習を成功させるには、学習信号とアーキテクチャを同時に設計する必要がある、という教訓を明確に示した点にある。
